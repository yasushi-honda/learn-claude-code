# セキュリティ設計

> 対応する公式ドキュメント: [Security](https://code.claude.com/docs/en/security)

## 学習目標

- Claude Codeのセキュリティモデルとパーミッションベースアーキテクチャを理解する
- コード実行の安全性と書き込みアクセス制限の仕組みを把握する
- プロンプトインジェクション攻撃とその対策を理解する
- MCP・IDE・クラウド実行環境のセキュリティ特性を知る
- セキュリティのベストプラクティスを個人・チームで実践できる

## 概要

Claude Codeのセキュリティは、Anthropicの包括的なセキュリティプログラムに基づいて設計されています。パーミッションベースのアーキテクチャ・組み込みのプロンプトインジェクション対策・透明性の高い操作フローが主要な特徴です。

セキュリティ認証情報（SOC 2 Type 2報告書・ISO 27001証明書等）は[Anthropic Trust Center](https://trust.anthropic.com)で確認できます。

## 本文

### パーミッションベースアーキテクチャ

Claude Codeはデフォルトで**読み取り専用パーミッション**を使用します。追加のアクション（ファイル編集・テスト実行・コマンド実行）が必要な場合、Claude Codeは明示的な許可を要求します。

**パーミッションフロー:**
1. Claudeがアクションを提案
2. ユーザーが承認（一度だけ or 自動承認リストに追加）
3. アクションが実行される

この設計により、ユーザーは常に制御下に置かれます。

### 書き込みアクセス制限

Claude Codeは、**起動したフォルダとそのサブフォルダ**にのみ書き込み可能です。親ディレクトリへの書き込みは明示的な許可なしには不可能です。

```
/home/user/
├── projects/
│   └── my-app/     ← Claude Code起動ディレクトリ
│       ├── src/    ← 書き込み可能
│       └── tests/  ← 書き込み可能
└── documents/      ← 書き込み不可（親ディレクトリ）
```

読み取りはより広い範囲で可能（システムライブラリ・依存関係へのアクセスに必要）ですが、書き込みはプロジェクトスコープに制限されます。

### 組み込みの保護機能

#### サンドボックスBashツール

```
/sandbox
```

Bashコマンドをファイルシステムとネットワーク分離でサンドボックス化します。詳細は[サンドボックス](02-sandboxing.md)を参照してください。

#### プロンプト疲労の軽減

よく使う安全なコマンドを許可リストに追加することで、承認の繰り返しを避けます：

```json
// settings.json
{
  "permissions": {
    "allow": ["Bash(npm run lint)", "Bash(git diff *)", "Bash(git status)"]
  }
}
```

#### Accept Editsモード

複数の編集を一括承認しながら、副作用のあるコマンドは引き続き確認が必要なモード。

### プロンプトインジェクション対策

プロンプトインジェクションは、攻撃者がAIアシスタントの指示を悪意のあるテキストで上書き・操作しようとする手法です。

#### コア保護

**パーミッションシステム:**
センシティブな操作には明示的な承認が必要です。

**コンテキスト認識分析:**
リクエスト全体を分析して潜在的に有害な指示を検出します。

**入力のサニタイズ:**
ユーザー入力を処理してコマンドインジェクションを防ぎます。

**コマンドブロックリスト:**
デフォルトで`curl`・`wget`等の任意のWebコンテンツを取得するリスクの高いコマンドをブロックします。

**ネットワークリクエストの承認:**
ネットワークリクエストを行うツールはデフォルトでユーザー承認が必要です。

**分離されたコンテキストウィンドウ:**
Webフェッチは分離されたコンテキストウィンドウを使用し、悪意のあるプロンプトのインジェクションを防ぎます。

**トラスト検証:**
初回のコードベース実行や新しいMCPサーバーはトラスト検証が必要です。

**コマンドインジェクション検出:**
不審なBashコマンドは許可リストに追加済みでも手動承認が必要になります。

#### プライバシー保護機能

- センシティブ情報の限定された保持期間
- ユーザーセッションデータへのアクセス制限
- データトレーニング設定のユーザー制御（コンシューマーユーザーは[プライバシー設定](https://claude.ai/settings/privacy)でいつでも変更可能）

### 不審なコンテンツを扱う際のベストプラクティス

1. 承認前にすべての提案コマンドをレビューする
2. 信頼できないコンテンツをClaudeに直接パイプしない
3. 重要なファイルへの提案変更を検証する
4. 外部Webサービスとのインタラクション時はVMを使用する
5. 不審な動作を`/bug`で報告する

### MCP（Model Context Protocol）のセキュリティ

Claude Codeはユーザーが設定したMCPサーバーとの連携を許可します。設定はソースコードの一部として管理され、エンジニアがバージョン管理にチェックインします。

**ガイドライン:**
- 自分で書いたMCPサーバー、または信頼できるプロバイダーのものを使用する
- Claude Codeのパーミッションでキャプすることが可能
- AnthropicはMCPサーバーの監査・管理は行っていない

### IDE セキュリティ

VS Code内でClaude Codeを実行する場合の詳細は[VS Codeセキュリティとプライバシー](https://code.claude.com/docs/en/vs-code#security-and-privacy)を参照してください。

### クラウド実行環境のセキュリティ

Claude Code on the web（クラウド実行）では追加のセキュリティ制御が適用されます：

- **分離されたVM**: 各クラウドセッションがAnthropicが管理する独立したVMで実行
- **ネットワークアクセス制御**: デフォルトでネットワークアクセスを制限、特定ドメインのみ許可可能
- **認証情報の保護**: スコープが絞られた認証情報を使用し、実際のGitHub認証トークンには変換して使用
- **ブランチ制限**: gitプッシュは現在の作業ブランチのみに制限
- **監査ログ**: クラウド環境内のすべての操作がコンプライアンスと監査のために記録
- **自動クリーンアップ**: セッション完了後にクラウド環境が自動的に終了

### セキュリティのベストプラクティス

#### センシティブなコードを扱う場合

- すべての提案変更を承認前にレビュー
- センシティブなリポジトリにはプロジェクト固有のパーミッション設定を使用
- 追加の分離のために[devcontainers](https://code.claude.com/docs/en/devcontainer)の使用を検討
- `/permissions`でパーミッション設定を定期的に監査

#### チームのセキュリティ

- [管理設定](05-managed-settings.md)で組織標準を強制
- バージョン管理で承認済みのパーミッション設定を共有
- チームメンバーにセキュリティのベストプラクティスをトレーニング
- [OpenTelemetryメトリクス](https://code.claude.com/docs/en/monitoring-usage)でClaude Codeの使用を監視
- [`ConfigChange`フック](https://code.claude.com/docs/en/hooks#configchange)でセッション中の設定変更を監査・ブロック

### セキュリティ脆弱性の報告

Claude Codeのセキュリティ脆弱性を発見した場合：

1. 公開せずに報告する
2. [HackerOneプログラム](https://hackerone.com/anthropic-vdp/reports/new?type=team&report_type=vulnerability)を通じて報告
3. 詳細な再現手順を含める
4. 公開前に対応のための時間を与える

### Windowsでの注意事項

Windows上でClaude Codeを実行する場合、WebDAVの有効化や`\\*`のようなパスへのアクセスを許可しないことを推奨します。WebDAVはMicrosoftによってセキュリティリスクを理由に非推奨化されており、Claude Codeがリモートホストへのネットワークリクエストを発行し、パーミッションシステムを回避する可能性があります。

## まとめ

- Claude Codeはデフォルトで読み取り専用パーミッションを使用し、追加アクションには明示的な許可が必要
- 書き込みアクセスは起動ディレクトリとそのサブフォルダに厳格に制限される
- プロンプトインジェクション対策として、コンテキスト認識分析・コマンドブロックリスト・分離されたコンテキストウィンドウが機能する
- MCPサーバーは信頼できるプロバイダーのものを使用し、パーミッションを適切に設定する
- クラウド実行環境では追加のセキュリティ制御（VM分離・監査ログ・自動クリーンアップ）が適用される
- チームでは管理設定と`/permissions`による定期監査でセキュリティポリシーを維持する
