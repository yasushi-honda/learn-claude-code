# 対話モードの使い方

> **対応公式ドキュメント**: https://code.claude.com/docs/en/interactive-mode
> **想定所要時間**: 約60分
> **難易度**: ★★☆☆☆

## この章の学習目標

この章を終えると、以下のことができるようになります：

1. 主要なキーボードショートカットを覚えて操作を効率化できる
2. マルチライン入力・Bash モード・Vim モードを使いこなせる
3. `Shift+Tab` でパーミッションモードを切り替えて安全に作業できる
4. `@` メンションでファイル・URL・画像・MCP リソースを参照できる
5. `/` コマンドで Claude Code の各機能にすばやくアクセスできる

---

## 1. キーボードショートカット

Claude Code の対話モードは、豊富なキーボードショートカットで操作を効率化できるように設計されています。すべてを一度に覚える必要はありません。まず「全般操作」を押さえ、慣れてきたらテキスト編集やモード切り替えのショートカットを追加していきましょう。

> **macOS ユーザーへ**: `Alt+B`、`Alt+F`、`Alt+Y`、`Alt+M`、`Alt+P` などのショートカットを使うには、ターミナルの Option キーを Meta として設定する必要があります。
> - **iTerm2**: Settings → Profiles → Keys → Left/Right Option key を "Esc+" に設定
> - **Terminal.app**: Settings → Profiles → Keyboard → "Use Option as Meta Key" にチェック

### 全般操作

| ショートカット | 説明 | 補足 |
|---|---|---|
| `Ctrl+C` | 現在の入力または生成をキャンセル | 標準の割り込みシグナル |
| `Ctrl+F` | バックグラウンドエージェントをすべて停止 | 3秒以内に2回押す |
| `Ctrl+D` | Claude Code セッションを終了 | EOF シグナル |
| `Ctrl+G` | デフォルトのテキストエディタで開く | プロンプトやカスタム返答を編集 |
| `Ctrl+L` | ターミナル画面をクリア | 会話履歴は保持される |
| `Ctrl+O` | verbose（詳細出力）のトグル | ツール使用と実行の詳細を表示 |
| `Ctrl+R` | コマンド履歴の逆検索 | 前のコマンドをインタラクティブに検索 |
| `Ctrl+V` / `Cmd+V` | クリップボードから画像を貼り付け | `Cmd+V` は iTerm2、`Alt+V` は Windows |
| `Ctrl+B` | 実行中のタスクをバックグラウンドへ | Bash コマンドとエージェントをバックグラウンド化 |
| `Ctrl+T` | タスクリストのトグル | ターミナルステータスエリアに表示（最大10タスク） |
| `←` / `→` | ダイアログのタブ切り替え | 権限ダイアログとメニューのナビゲーション |
| `↑` / `↓` | コマンド履歴のナビゲート | 前の入力を呼び出す |
| `Esc` + `Esc` | 巻き戻しまたは要約 | コードや会話を前の状態に戻す |
| `Shift+Tab` / `Alt+M` | パーミッションモードのトグル | Auto-Accept・Plan・通常モードを循環 |
| `Option+P` / `Alt+P` | モデルの切り替え | プロンプトをクリアせずに切り替え |
| `Option+T` / `Alt+T` | 拡張思考のトグル | 拡張思考モードの有効/無効（要 `/terminal-setup`） |

### テキスト編集

| ショートカット | 説明 | 補足 |
|---|---|---|
| `Ctrl+K` | カーソルから行末まで削除 | 削除したテキストを `Ctrl+Y` で貼り付け可能 |
| `Ctrl+U` | 行全体を削除 | 削除したテキストを `Ctrl+Y` で貼り付け可能 |
| `Ctrl+Y` | 削除したテキストを貼り付け | `Ctrl+K` や `Ctrl+U` で削除したテキスト |
| `Alt+Y` | 貼り付け履歴を循環 | `Ctrl+Y` 後に使用 |
| `Alt+B` | カーソルを1単語後退 | 単語単位のナビゲーション |
| `Alt+F` | カーソルを1単語前進 | 単語単位のナビゲーション |

> **公式ドキュメントより**: これらのショートカットは Emacs スタイルのキーバインドに基づいています。Vim スタイルを好む場合は `/vim` コマンドで切り替えられます。

## 2. 入力モードとクイックアクセス

### マルチライン入力

複数行のプロンプトを入力する方法は複数あります。ターミナル環境に応じて使い分けてください：

| 方法 | ショートカット | 対応ターミナル |
|---|---|---|
| クイックエスケープ | `\` + `Enter` | すべてのターミナル（最も互換性が高い） |
| macOS デフォルト | `Option+Enter` | macOS デフォルト |
| Shift+Enter | `Shift+Enter` | iTerm2・WezTerm・Ghostty・Kitty で設定不要 |
| 制御シーケンス | `Ctrl+J` | マルチライン用ラインフィード |
| 貼り付けモード | 直接貼り付け | コードブロック・ログなどの大きなテキスト |

> **ヒント**: Shift+Enter は iTerm2・WezTerm・Ghostty・Kitty では設定なしで動作します。VS Code・Alacritty・Zed・Warp では `/terminal-setup` を実行してバインドをインストールしてください。

### クイックプレフィックス

Claude Code には3つのクイックアクセスプレフィックスがあります。通常の自然言語入力と組み合わせることで、操作がスムーズになります：

| プレフィックス | 機能 | 例 |
|---|---|---|
| `/` | ビルトインコマンドを実行 | `/clear`、`/model opus`、`/diff` |
| `!` | Bash コマンドを直接実行 | `! npm test`、`! git status` |
| `@` | ファイル・リソースを参照 | `@src/auth.ts`、`@github:repos/...` |

### Bash モード（`!` プレフィックス）

`!` で始めると Claude を介さずに Bash コマンドを直接実行できます：

```bash
! npm test
! git status
! ls -la
```

実行結果は会話コンテキストに自動追加されます。長時間かかるコマンドは `Ctrl+B` でバックグラウンドに移行できます。

### プロンプトサジェスション

Claude Code は入力中にプロンプトのサジェスションを表示することがあります。`Tab` キーで受理できます。この機能を無効にするには環境変数を設定します：

```bash
export CLAUDE_CODE_ENABLE_PROMPT_SUGGESTION=false
```

## 3. パーミッションモードの切り替え

`Shift+Tab`（または `Alt+M`）を押すとパーミッションモードが循環します。作業内容に応じて適切なモードを選ぶことが、安全かつ効率的な開発の鍵です。

```
通常モード（Default）
    ↓ Shift+Tab
Auto-Accept モード（⏵⏵ accept edits on）
    ↓ Shift+Tab
Plan モード（⏸ plan mode on）
    ↓ Shift+Tab
通常モード（Default）
```

| モード | 動作 | 推奨場面 |
|---|---|---|
| **Default** | ファイル編集とシェルコマンドの前に確認を求める | 慎重に進めたいとき（デフォルト） |
| **Auto-accept edits** | ファイル編集は確認なし、コマンドは確認あり | 信頼できる編集を連続実行するとき |
| **Plan mode** | 読み取り専用ツールのみ使用、変更前にプランを提示 | 設計段階・コードレビュー・分析 |

> **公式ドキュメントより**: Plan mode は CLI からも `claude --permission-mode plan` で起動できます。`Ctrl+G` を押すとプランをデフォルトのテキストエディタで開いて直接編集できます。

## 4. @メンション：ファイル・URL・画像の参照

`@` を入力するとファイルパスのオートコンプリートが起動します。ファイルの内容や構造を会話に直接含めることで、Claude がより正確に理解できます。

### ファイルを参照する

```
> Explain the logic in @src/utils/auth.js
```

ファイルの全内容が会話に含まれます。

### ディレクトリを参照する

```
> What's the structure of @src/components?
```

ファイル情報付きのディレクトリ一覧が提供されます（ファイルの内容ではなく一覧）。

### MCP リソースを参照する

```
> Show me the data from @github:repos/owner/repo/issues
```

接続された MCP サーバーからデータを取得します（フォーマット: `@server:resource`）。

**重要なポイント:**

- ファイルパスは相対パス・絶対パスどちらでも可
- `@` ファイル参照は、ファイルのディレクトリと親ディレクトリの `CLAUDE.md` もコンテキストに追加する
- 1つのメッセージで複数ファイルを参照可能（例: `@file1.js and @file2.js`）
- 画像の参照は `Cmd+Click`（Mac）または `Ctrl+Click`（Windows/Linux）でデフォルトビューアで開ける

## 6. Vim エディタモードとバックグラウンドタスク

### Vim モード

`/vim` コマンドまたは `/config` で永続的に Vim スタイル編集を有効にできます。

**モード切り替え:**

| コマンド | アクション | 元のモード |
|---|---|---|
| `Esc` | NORMAL モードへ | INSERT |
| `i` | カーソル前に挿入 | NORMAL |
| `I` | 行頭に挿入 | NORMAL |
| `a` | カーソル後に挿入 | NORMAL |
| `A` | 行末に挿入 | NORMAL |

**NORMAL モードのナビゲーション:**

| コマンド | アクション |
|---|---|
| `h`/`j`/`k`/`l` | 左/下/上/右に移動 |
| `w` / `e` / `b` | 次の単語 / 単語の末尾 / 前の単語 |
| `0` / `$` | 行頭 / 行末 |
| `gg` / `G` | 入力の先頭 / 末尾 |

**編集コマンド:**

| コマンド | アクション |
|---|---|
| `x` | カーソル下の文字を削除 |
| `dd` | 行全体を削除 |
| `dw` | 単語を削除 |
| `cw` | 単語を変更（削除して INSERT へ） |
| `cc` | 行を変更 |
| `yy` | 行をヤンク（コピー） |
| `p` | 貼り付け |
| `u` | 元に戻す |

**テキストオブジェクト:**

| コマンド | アクション |
|---|---|
| `diw` | 内側の単語を削除 |
| `ciw` | 内側の単語を変更 |
| `di"` | ダブルクォート内を削除 |
| `ci(` | 括弧内を変更 |

### バックグラウンドタスク

長時間かかる Bash コマンドやエージェントの実行中に `Ctrl+B` を押すと、タスクをバックグラウンドに移行できます。バックグラウンドで実行中のタスクは `Ctrl+T` でタスクリストを表示して確認できます。

タスクリストの特徴：
- ターミナルステータスエリアに表示される
- 最大10タスクまで同時管理可能
- 完了したタスクの結果は会話に自動追加される

バックグラウンドタスクを無効化するには環境変数を設定します：

```bash
export CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1
```

### コマンド履歴

Claude Code はセッション中のコマンド履歴を管理します：

- `↑` / `↓` で前の入力を呼び出す
- `Ctrl+R` で逆順検索（検索ワードを入力し、`Ctrl+R` でさらに古いマッチを循環）
- 入力履歴はプロジェクトの作業ディレクトリごとに保存
- `/clear` を実行すると現在のセッション履歴はリセット（会話は保存される）

### セッションピッカー

`/resume` またはセッション一覧画面では以下のショートカットが使えます：

| ショートカット | アクション |
|---|---|
| `↑` / `↓` | セッション選択を移動 |
| `→` / `←` | セッションの詳細を展開/折りたたみ |
| `Enter` | 選択したセッションを再開 |
| `P` | セッションにピンを付ける |
| `R` | セッション名を変更 |
| `/` | セッションを検索 |
| `A` | すべてのセッションを表示（現在のリポジトリ以外も含む） |
| `B` | ブランチでフィルタ |
| `Esc` | ピッカーを閉じる |

### PR Review ステータス表示

ブランチでオープンな PR がある場合、フッターにクリック可能な PR リンクが表示されます（例: "PR #446"）。色付きのアンダーラインでレビュー状態を示します：

- 緑: 承認済み
- 黄: レビュー待ち
- 赤: 変更要求あり
- グレー: ドラフト
- 紫: マージ済み

## ハンズオン演習

### 演習 1: キーボードショートカットを体験する

**目的**: 基本的なキーボードショートカットを実際に操作して覚える
**前提条件**: Claude Code がインストール済み

**手順**:
1. `claude` でセッションを開始する
2. 何か質問を入力し（例: `hello`）、生成途中で `Ctrl+C` を押してキャンセルする
3. `Ctrl+L` で画面をクリアする
4. `Ctrl+O` を押して verbose モードをオンにし、質問して出力を確認する。もう一度 `Ctrl+O` で戻す
5. `/status` でセッション情報を確認する
6. `/cost` でトークン使用量を確認する

**期待される結果**: 各ショートカットの動作を体感し、verbose 出力でツール使用の詳細が見えることを確認

### 演習 2: @メンションとマルチライン入力

**目的**: ファイル参照とマルチライン入力をマスターする
**前提条件**: 任意のプロジェクトディレクトリで Claude Code を起動

**手順**:
1. `@` を入力してオートコンプリートを確認する
2. プロジェクト内の任意のファイルを `@ファイル名` で参照し、「このファイルを説明して」と聞く
3. `\` + `Enter` でマルチライン入力を試す（2行以上のプロンプトを入力）
4. ディレクトリを `@ディレクトリ名` で参照し、構造を確認する

**期待される結果**: `@` メンションでファイルの内容がコンテキストに追加され、Claude がその内容に基づいて回答する

### 演習 3: パーミッションモードの切り替え

**目的**: 3つのモードの違いを理解する
**前提条件**: 演習 1 のセッションを継続するか新しいセッションを開始

**手順**:
1. `Shift+Tab` を押して Auto-accept モードに切り替え、フッターの表示を確認する
2. もう一度 `Shift+Tab` を押して Plan モードに切り替える
3. Plan モードで `create a hello.txt file` と指示し、ファイルが作成されない（プランのみ提示される）ことを確認する
4. `Shift+Tab` で通常モードに戻す

**期待される結果**: 各モードでフッターの表示が変わり、Plan モードではファイル変更が実行されないことを確認

## よくある質問

**Q: Shift+Enter がターミナルで動作しません**
A: VS Code・Alacritty・Zed・Warp では `/terminal-setup` を実行してキーバインドをインストールしてください。代替として `\` + `Enter` はすべてのターミナルで動作します。

**Q: Option+T を押しても拡張思考が切り替わりません**
A: `/terminal-setup` を実行してキーバインドを設定する必要があります。また、ターミナルで Option キーが Meta として設定されていることを確認してください。

**Q: /コマンドの一覧に表示されないコマンドがあります**
A: プランやプラットフォームによって利用可能なコマンドが異なります。`/help` で現在利用可能なコマンドを確認できます。

**Q: Ctrl+B でバックグラウンドに移行したタスクの結果はどうなりますか？**
A: タスクが完了すると結果は自動的に会話コンテキストに追加されます。`Ctrl+T` でタスクリストを表示して進行状況を確認できます。

**Q: Vim モードは永続的に設定できますか？**
A: `/config` から永続設定にすることも、`/vim` でセッション単位でトグルすることもできます。

## まとめ

この章で学んだ重要ポイント：

- `Ctrl+C`（キャンセル）、`Ctrl+L`（画面クリア）、`Ctrl+O`（verbose）、`Ctrl+B`（バックグラウンド化）が最頻出のショートカット
- マルチライン入力は `\` + `Enter` が最も互換性が高い
- `Shift+Tab` でパーミッションモードを切り替え（Default → Auto-accept → Plan）、作業内容に応じて使い分ける
- `@` でファイル・ディレクトリ・MCP リソースをコンテキストに追加。`/` でコマンド実行、`!` で Bash 直接実行
- Vim モード・バックグラウンドタスク・タスクリストなど、上級者向けの機能も充実している
