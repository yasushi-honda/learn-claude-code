# 出力スタイル設定

> **対応公式ドキュメント**: https://code.claude.com/docs/en/output-styles
> **想定所要時間**: 約60分
> **難易度**: ★★☆☆☆

## この章の学習目標

この章を終えると、以下のことができるようになります：

1. ビルトインの出力スタイル（Default、Explanatory、Learning）の特徴を理解し、使い分けられる
2. `/output-style` コマンドで出力スタイルを変更できる
3. カスタム出力スタイルを作成して、Claude Code を様々なエージェントとして活用できる
4. `--output-format` フラグでテキスト/JSON/ストリーミング JSON 出力を制御できる

---

## 1. 出力スタイルとは

出力スタイルは、Claude Code の応答の仕方を根本的に変える機能です。デフォルトはソフトウェアエンジニアリングタスクに最適化されていますが、スタイルを変更することで教育モードやカスタムエージェントとして機能させることができます。

> **公式ドキュメントより**: 出力スタイルは Claude Code の**システムプロンプトを直接変更**します。CLAUDE.md がユーザーメッセージとして追加されるのとは異なり、システムプロンプト自体が書き換わります。

---

## 2. ビルトインの出力スタイル

### Default（デフォルト）

既存のシステムプロンプトで、ソフトウェアエンジニアリングタスクを効率的にこなすよう設計されています。簡潔な応答を優先します。

### Explanatory（解説型）

```
/output-style explanatory
```

コーディングタスクの合間に教育的な「インサイト」を提供します。

**特徴**:
- 変更を加えながらその理由を説明する
- コードベースのパターンを解説する
- コーディング規約を教育的に提示する

**推奨シーン**: 新しいコードベースに参加したとき、ジュニアエンジニアの教育

### Learning（学習型）

```
/output-style learning
```

一緒にコードを書いて学ぶモードです。

**特徴**:
- コードに `TODO(human)` マーカーを挿入してあなたに実装を促す
- 解説しながらインサイトを共有する
- 学習効果を最大化する段階的な実装

**推奨シーン**: プログラミング学習、ペアプログラミング体験

---

## 3. 出力スタイルの変更方法

### コマンドメニューから変更

```
/output-style
```

メニューが表示され、利用可能なスタイルを選択できます。`/config` メニューからもアクセスできます。

### 直接指定して変更

```
/output-style explanatory
/output-style learning
/output-style default
```

### 設定ファイルで指定

```json
{
  "outputStyle": "Explanatory"
}
```

変更はローカルプロジェクトレベルで保存され、`.claude/settings.local.json` に記録されます。

## 4. カスタム出力スタイルの作成

### ファイルの保存場所

| スコープ | パス | 適用範囲 |
|---------|-----|---------|
| ユーザーレベル | `~/.claude/output-styles/` | 全プロジェクトで使用可能 |
| プロジェクトレベル | `.claude/output-styles/` | プロジェクト内のみ |

### ファイルフォーマット

カスタム出力スタイルはフロントマター付きの Markdown ファイルで定義します。

```markdown
---
name: My Custom Style
description:
  このスタイルが何をするかの簡単な説明（ユーザーに表示される）
keep-coding-instructions: false
---

# カスタムスタイルの指示

あなたはソフトウェアエンジニアリングタスクをサポートする CLI ツールです。
[カスタム指示をここに記述...]

## 具体的な動作

[このスタイルでの振る舞いを定義...]
```

### フロントマターのフィールド

| フィールド | 用途 | デフォルト |
|----------|------|----------|
| `name` | スタイルの名前（ファイル名と異なる場合） | ファイル名を継承 |
| `description` | `/output-style` UI に表示される説明 | なし |
| `keep-coding-instructions` | コーディング関連のシステムプロンプトを保持するか | `false` |

### keep-coding-instructions の動作

| 値 | 動作 |
|-----|------|
| `true` | デフォルトのコーディング指示（テストで検証する等）を保持し、カスタム指示を追加 |
| `false` | コーディング関連の指示を除外し、カスタム指示で置換 |

コーディングアシスタントとしてのスタイル（コードレビューアーなど）には `true`、コーディング以外の用途（テクニカルライターなど）には `false` を使います。

---

## 5. 出力スタイルの仕組み

出力スタイルは以下の手順でシステムプロンプトに影響します。

1. **全スタイル共通**: 効率的な出力のための指示（簡潔に応答する等）を除外
2. **カスタムスタイル**: `keep-coding-instructions` が `false` の場合、コーディング関連の指示を除外
3. **全スタイル共通**: カスタム指示をシステムプロンプトの末尾に追加
4. **全スタイル共通**: 会話中、スタイルの指示に従うようリマインダーを送信

### 関連機能との違い

| | CLAUDE.md | 出力スタイル | スキル |
|--|----------|-----------|------|
| **変更対象** | なし（ユーザーメッセージとして追加） | システムプロンプトを直接変更 | ワークフロー定義 |
| **適用時期** | セッション開始時に常時 | 選択後は常時 | 呼び出し時または関連ある場合のみ |
| **用途** | プロジェクト固有の指示 | 全体的な動作モード | 再利用可能なワークフロー |

### --append-system-prompt との違い

`--append-system-prompt` はデフォルトシステムプロンプトをそのまま保持してコンテンツを追加しますが、カスタム出力スタイルは（`keep-coding-instructions: false` の場合）コーディング固有の指示を除外してシステムプロンプトを置き換えます。

## 6. --output-format フラグ（CLI）

コマンドラインから出力フォーマットを指定できます。これは `--print`（`-p`）フラグと組み合わせて使います。

```bash
# テキスト形式（デフォルト）
claude -p "コードをレビューして" --output-format text

# JSON 形式
claude -p "コードをレビューして" --output-format json

# ストリーミング JSON
claude -p "コードをレビューして" --output-format stream-json
```

| フォーマット | 用途 |
|-----------|------|
| `text` | 通常の対話（デフォルト） |
| `json` | プログラムからの出力処理 |
| `stream-json` | リアルタイムのストリーミング処理 |

## 7. ソフトウェアエンジニアリング以外への活用

カスタム出力スタイルを使うことで、Claude Code を様々なエージェントとして活用できます。

### テクニカルライター

```markdown
---
name: Technical Writer
description: ドキュメントと技術文書の作成を支援
keep-coding-instructions: false
---

あなたは技術文書の専門ライターです。
明確で簡潔な説明を提供し、コードよりも文章での説明を優先します。
```

### コードレビューアー

```markdown
---
name: Code Reviewer
description: 詳細なコードレビューを提供
keep-coding-instructions: true
---

あなたは厳格なコードレビューアーです。
セキュリティ・パフォーマンス・保守性の観点から問題を指摘します。
変更の提案より、問題の説明を優先してください。
```

### アーキテクチャアドバイザー

```markdown
---
name: Architecture Advisor
description: システム設計とアーキテクチャの分析・助言
keep-coding-instructions: false
---

あなたはシステムアーキテクチャの専門家です。
設計パターン、スケーラビリティ、保守性の観点から分析と提案を行います。
コードの実装よりも、設計の全体像と意思決定を重視してください。
```

## ハンズオン演習

### 演習 1: ビルトインスタイルの違いを体験する

**目的**: Default、Explanatory、Learning の応答の違いを比較する

**前提条件**: 任意のプロジェクトで Claude Code が起動できること

**手順**:

1. Default スタイルで起動し、以下を依頼:
   ```
   src/ ディレクトリの構造を分析してください
   ```
   応答のスタイルを観察する

2. Explanatory に切り替え:
   ```
   /output-style explanatory
   ```
   同じ質問を再度依頼し、「インサイト」が含まれることを確認する

3. Learning に切り替え:
   ```
   /output-style learning
   ```
   「簡単な関数を作成してください」と依頼し、`TODO(human)` が含まれることを確認する

4. Default に戻す:
   ```
   /output-style default
   ```

**期待される結果**: 各スタイルで応答の形式と詳細さが大きく異なることを体験できる。

### 演習 2: カスタム出力スタイルを作成する

**目的**: 自分専用のカスタムスタイルを作成する

**手順**:

1. ユーザーレベルのディレクトリを作成:
   ```bash
   mkdir -p ~/.claude/output-styles
   ```

2. カスタムスタイルファイルを作成（`~/.claude/output-styles/mentor.md`）:
   ```markdown
   ---
   name: Mentor
   description: コーディングメンターとして段階的に教える
   keep-coding-instructions: true
   ---

   あなたはシニアエンジニアのメンターです。

   ## 動作方針
   - 答えをすぐに教えるのではなく、ヒントを出して考えさせる
   - なぜその実装が良いのか、背景と理由を説明する
   - コードを書く前に、設計の選択肢を提示する
   - 「こう書く方法もあります」と代替案を示す
   ```

3. Claude Code を起動し、カスタムスタイルを選択:
   ```
   /output-style mentor
   ```

4. 「ユーザー認証の関数を作成してください」と依頼し、メンターらしい応答を確認する

**期待される結果**: カスタムスタイルが `/output-style` のリストに表示され、指定した動作方針に従った応答が得られる。

### 演習 3: --output-format で JSON 出力を取得する

**目的**: CLI から JSON 形式で出力を取得し、スクリプトから活用する方法を習得する

**手順**:

1. テキスト形式で出力:
   ```bash
   claude -p "2 + 2 は？" --output-format text
   ```

2. JSON 形式で出力:
   ```bash
   claude -p "2 + 2 は？" --output-format json
   ```

3. JSON をパイプで処理:
   ```bash
   claude -p "このディレクトリのファイル一覧を教えて" --output-format json | jq '.result'
   ```

**期待される結果**: JSON 形式の出力が得られ、`jq` などのツールでプログラム的に処理できる。

## よくある質問

**Q: 出力スタイルの変更はセッションを超えて保持されますか？**
A: はい。`/output-style` で変更すると `.claude/settings.local.json` に保存され、次回のセッションでも維持されます。

**Q: CLAUDE.md の指示と出力スタイルの指示が矛盾する場合はどうなりますか？**
A: 出力スタイルはシステムプロンプトを変更し、CLAUDE.md はユーザーメッセージとして追加されます。一般的にはシステムプロンプトの指示が優先されますが、矛盾しないよう設計することが重要です。

**Q: keep-coding-instructions を false にすると何が変わりますか？**
A: コーディング関連のデフォルト指示（テストで検証する、簡潔に応答するなど）が除外されます。コーディング以外の用途（ライティング、分析など）で使う場合に `false` を指定します。

**Q: --output-format は対話モードでも使えますか？**
A: `--output-format` は主に `--print`（`-p`）フラグとの組み合わせ（非対話モード）で使います。対話モードでの出力フォーマットは出力スタイルで制御します。

## まとめ

- **3つのビルトインスタイル**: Default（効率的）、Explanatory（教育的インサイト）、Learning（協調学習）
