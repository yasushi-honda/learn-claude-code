# デスクトップアプリ

> 対応する公式ドキュメント: [Use Claude Code Desktop](https://code.claude.com/docs/en/desktop)

## 学習目標

- Claude Code デスクトップアプリの主要機能を理解する
- 並列セッションと Git worktree 分離の仕組みを把握する
- ビジュアルdiff 表示と PR 監視機能を使いこなす
- アプリプレビュー機能の設定方法を理解する
- `/desktop` コマンドでのハンドオフを習得する

## 概要

Claude Desktop アプリの「Code タブ」は、Claude Code をグラフィカルインターフェースで使用する方法を提供します。ターミナルの CLI に加えて以下の機能が使えます：

- **ビジュアルdiff レビュー**とインラインコメント
- **ライブアプリプレビュー**（開発サーバー連携）
- **GitHub PR 監視**（自動修正・自動マージ）
- **並列セッション**（Git worktree による自動分離）
- **コネクタ**（GitHub、Slack、Linear など）
- **ローカル・SSH・クラウド**の各環境対応

## セッションの開始

セッション開始前に、プロンプトエリアで4つの項目を設定します：

| 項目 | 説明 |
|-----|------|
| **環境** | ローカル（自分のマシン）、リモート（Anthropic クラウド）、SSH |
| **プロジェクトフォルダ** | Claude が作業するフォルダまたはリポジトリ |
| **モデル** | 送信ボタン隣のドロップダウンでモデルを選択 |
| **パーミッションモード** | Claude の自律性の度合いを設定 |

## パーミッションモード

| モード | 説明 | 推奨シーン |
|-------|------|----------|
| **確認を求める（Ask permissions）** | ファイル編集・コマンド実行の前に都度確認 | 初回利用時・重要な作業 |
| **編集を自動承認（Auto accept edits）** | ファイル編集は自動承認、コマンド実行は確認 | ファイル変更を信頼している場合 |
| **プランモード（Plan mode）** | コードを分析して計画を立て、変更はしない | 複雑なタスクの事前確認 |
| **全バイパス（Bypass permissions）** | 全ての承認プロンプトをスキップ | サンドボックス環境のみ |

> **推奨**: 複雑なタスクはプランモードで始め、計画を承認してから自動承認か通常モードで実行します。

## 並列セッション（Git worktree 分離）

サイドバーの **+ New session** をクリックすることで、複数のタスクを並行して実行できます。

Git リポジトリでは、各セッションに Git worktree による**独立したファイルコピー**が自動的に割り当てられます。これにより、あるセッションでの変更が他のセッションに影響しません。

### Worktree の設定

- デフォルト保存場所: `<project-root>/.claude/worktrees/`
- 設定 → Claude Code → 「Worktree location」でカスタムディレクトリに変更可能
- ブランチプレフィックスの設定で Claude が作成するブランチを整理できる
- セッション完了後、サイドバーでホバーしてアーカイブアイコンをクリックで削除

> **注意**: Git セッション分離には Git が必要です。Windows では必須。

## アプリプレビュー

Claude が開発サーバーを起動して、変更をブラウザプレビューで確認できます。

### 動作

- フロントエンドウェブアプリおよびバックエンドサーバーに対応
- 通常、Claude はファイル編集後に自動的にサーバーを起動
- プレビューパネルから直接アプリを操作できる

### 自動検証（autoVerify）

`autoVerify` が有効の場合、Claude はファイル編集後に自動的に変更を検証します：
- スクリーンショットを撮る
- DOM を確認する
- 要素をクリック・フォーム入力して動作確認
- 問題があれば自動修正

#### 設定ファイル（.claude/launch.json）

```json
{
  "version": "0.0.1",
  "configurations": [
    {
      "name": "my-app",
      "runtimeExecutable": "npm",
      "runtimeArgs": ["run", "dev"],
      "port": 3000
    }
  ]
}
```

`autoVerify` を無効にするには：

```json
{
  "version": "0.0.1",
  "autoVerify": false,
  "configurations": [...]
}
```

### 設定フィールド

| フィールド | 説明 |
|----------|------|
| `name` | サーバーの識別名 |
| `runtimeExecutable` | 実行コマンド（`npm`、`yarn`、`node` など） |
| `runtimeArgs` | コマンドへの引数（例: `["run", "dev"]`） |
| `port` | サーバーのポート番号（デフォルト: 3000） |
| `cwd` | プロジェクトルートからの相対作業ディレクトリ |
| `env` | 追加の環境変数 |
| `autoPort` | ポート競合時の対処（true: 自動、false: エラー） |

## ビジュアルdiff レビュー

Claude がコードを変更すると、変更行数のインジケーター（例: `+12 -1`）が表示されます。

1. インジケーターをクリック → diff ビューアが開く
2. 左側にファイルリスト、右側に各ファイルの変更内容が表示
3. 特定の行をクリックしてコメントを追加できる
4. 複数行にコメントを追加後、`Cmd+Enter`（Mac）/ `Ctrl+Enter`（Windows）で一括送信
5. Claude はコメントを読んで変更を加え、新しいdiff が表示される

## PR 監視

プルリクエストを作成すると、セッション内に CI ステータスバーが表示されます。

| 機能 | 説明 |
|-----|------|
| **自動修正（Auto-fix）** | CI が失敗すると Claude が自動的に修正を試みる |
| **自動マージ（Auto-merge）** | 全チェックが通ると自動マージ（スカッシュ方式） |

> **注意**: PR 監視には GitHub CLI（`gh`）のインストールと認証が必要です。自動マージはリポジトリの設定でも有効化が必要。

## /desktop コマンド（ターミナルからハンドオフ）

ターミナルでの CLI セッションをデスクトップアプリに移行できます：

```bash
/desktop
```

このコマンドを実行すると：
1. セッションを保存してデスクトップアプリで開く
2. CLI を終了する

> macOS と Windows のみ対応。

## 外部ツールのコネクタ

ローカルおよび SSH セッションでは、プロンプトボックスの **+** ボタン → **Connectors** から外部サービスを接続できます：

- Google Calendar
- Slack
- GitHub
- Linear
- Notion
- その他多数

コネクタを接続すると、Claude がカレンダーの読み取り、メッセージ送信、Issue 作成などを直接行えます。

## CLI との比較

| 機能 | CLI | デスクトップ |
|-----|-----|------------|
| 全パーミッションモード | 対応（`dontAsk` を含む） | Ask、Auto accept、Plan、Bypass |
| サードパーティプロバイダー | 対応 | 非対応（Anthropic API 直接接続のみ） |
| MCP サーバー設定 | 設定ファイル | コネクタ UI または設定ファイル |
| ファイル添付 | 非対応 | 画像・PDF |
| セッション分離 | `--worktree` フラグ | 自動 worktree |
| スクリプト・自動化 | `--print` フラグなど | 非対応 |

## まとめ

- デスクトップアプリは CLI と同じエンジンでグラフィカルに操作できる
- 並列セッションは Git worktree で自動的に分離される
- プランモードで計画を確認してから実行するのがベストプラクティス
- アプリプレビューで開発サーバーの動作を確認できる
- PR 監視で CI 失敗の自動修正と自動マージが可能
- `/desktop` でターミナルセッションをデスクトップアプリに移行できる
