# ターミナル最適化

> **対応公式ドキュメント**: https://code.claude.com/docs/en/terminal-config
> **想定所要時間**: 約60分
> **難易度**: ★★☆☆☆

## この章の学習目標

この章を終えると、以下のことができるようになります：

1. Claude Code に最適なターミナルのテーマと外観を設定できる
2. 改行入力の複数の方法を理解し、自分のターミナルに合った方法を選択できる
3. デスクトップ通知を設定して、Claude Code の完了を見逃さなくなる
4. 長い入力を安全に扱うファイルベースワークフローを実践できる
5. Vim モードを有効化し、Vim キーバインドで入力を操作できる

---

## 1. テーマと外観のカスタマイズ

Claude Code はターミナル上で動作するため、見た目はターミナルアプリケーションの設定に大きく依存します。フォントやカラースキームはターミナルアプリ側で設定しますが、Claude Code 自体の UI テーマは `/config` コマンドで変更できます。

### テーマの変更

```
/config
```

`/config` を実行すると設定画面が開き、テーマ（ライト / ダーク / ターミナルに合わせる）を選択できます。

### カスタムステータスライン

ターミナル下部に表示されるステータスラインには、現在使用中のモデル、作業ディレクトリ、Git ブランチなどの情報を表示できます。ステータスラインのカスタマイズ方法は Week 4 の「出力スタイル設定」で詳しく扱います。

> **公式ドキュメントより**: ステータスラインのカスタマイズにより、セッションの状態を一目で把握でき、コンテキストスイッチのコストを下げられます。

## 2. 改行入力の設定

Claude Code でプロンプトを入力するとき、改行を含む複数行のプロンプトを書きたい場面は頻繁にあります。Enter キーはプロンプトの「送信」に割り当てられているため、改行を入力するには別の方法が必要です。

### 方法 1: バックスラッシュ + Enter（全ターミナル共通）

最も汎用的な方法です。どのターミナルでも使えます。

```
\ + Enter
```

`\` を入力した直後に Enter を押すと、送信されずに改行が挿入されます。

### 方法 2: Shift+Enter（ネイティブサポートのターミナル）

以下のターミナルは追加設定なしで Shift+Enter による改行入力が使えます：

| ターミナル | 対応状況 |
|-----------|---------|
| **iTerm2** | 設定不要で対応 |
| **WezTerm** | 設定不要で対応 |
| **Ghostty** | 設定不要で対応 |
| **Kitty** | 設定不要で対応 |

### 方法 3: `/terminal-setup` による自動設定

上記以外のターミナルでは、Claude Code 内で `/terminal-setup` コマンドを実行することで Shift+Enter を自動設定できます。

```
/terminal-setup
```

自動設定に対応するターミナル：

| ターミナル | 設定方法 |
|-----------|---------|
| **VS Code** | `/terminal-setup` で自動設定 |
| **Alacritty** | `/terminal-setup` で自動設定 |
| **Zed** | `/terminal-setup` で自動設定 |
| **Warp** | `/terminal-setup` で自動設定 |

> **公式ドキュメントより**: `/terminal-setup` コマンドは、設定が必要なターミナルでのみ表示されます。iTerm2、WezTerm、Ghostty、Kitty ではこのコマンドは表示されません（既にネイティブで対応しているため）。

### Option+Enter の設定

一部のターミナルでは Option+Enter（Alt+Enter）による改行入力も使えますが、追加設定が必要です。

#### macOS Terminal.app

1. **設定** → **プロファイル** → **キーボード** を開く
2. 「**Use Option as Meta Key**（Option キーをメタキーとして使用）」にチェックを入れる

#### iTerm2

1. **Settings** → **Profiles** → **Keys** を開く
2. **General** タブで、**Left Option key** を「**Esc+**」に設定
3. 必要に応じて **Right Option key** も同様に設定

#### VS Code ターミナル

1. **Settings** → **Terminal** の設定を開く
2. **Left Option key** および **Right Option key** を「**Esc+**」に設定

## 3. デスクトップ通知の設定

Claude Code が処理を完了して入力待ちになると、通知イベントが発生します。この通知をデスクトップ通知として表示することで、長い処理が終わった瞬間に気づけるようになります。

### ターミナル別の対応状況

| ターミナル | 対応状況 | 設定方法 |
|-----------|---------|---------|
| **Kitty** | デフォルトで対応 | 追加設定なし |
| **Ghostty** | デフォルトで対応 | 追加設定なし |
| **iTerm2** | 設定が必要 | 下記参照 |
| **macOS Terminal.app** | ネイティブ非対応 | Notification hooks を使用 |
| **その他** | ネイティブ非対応 | Notification hooks を使用 |

### iTerm2 での通知設定

1. **iTerm2** の **Settings** → **Profiles** → **Terminal** を開く
2. 「**Notification Center Alerts**（通知センターアラート）」を有効にする
3. 「**Filter Alerts**（アラートをフィルタ）」をクリックし、「**Send alerts generated by escape sequences**（エスケープシーケンスで生成されたアラートを送信）」にチェックを入れる

通知が表示されない場合は、macOS の **システム設定** → **通知** でターミナルアプリに通知権限が付与されているか確認してください。

### Notification hooks によるカスタム通知

ネイティブ通知に対応しないターミナルや、カスタムの通知動作（サウンド再生、Slack へのメッセージ送信など）を設定したい場合は、Notification hooks を使います。

hooks は `.claude/settings.json` に記述します。例えば、通知時にサウンドを再生する設定：

```json
{
  "hooks": {
    "Notification": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "afplay /System/Library/Sounds/Glass.aiff"
          }
        ]
      }
    ]
  }
}
```

> **公式ドキュメントより**: hooks はターミナル通知と並行して動作します。ネイティブ通知が表示されるターミナルでも、追加の hooks を設定可能です。

## 4. 長い入力の扱い方

大量のコードや長い指示文を Claude Code に渡したい場面があります。しかし、長いテキストの直接貼り付けにはいくつかの問題があります。

### 直接貼り付けの問題点

| 問題 | 説明 |
|-----|------|
| **処理の不正確さ** | 非常に長いコンテンツを貼り付けると、Claude Code が正しく処理できない場合がある |
| **VS Code の制限** | VS Code のターミナルは長い貼り付けを途中で切り捨てることがある |
| **ターミナルのバッファ制限** | 一部のターミナルには入力バッファの上限がある |

### ファイルベースワークフロー（推奨）

長い入力は、ファイルに書き出してから Claude Code にそのファイルを読ませるアプローチが推奨されます。

```bash
# ステップ 1: 長い指示をファイルに書き出す
cat > /tmp/my-instructions.md << 'EOF'
以下の処理を実装してください：

1. ユーザー認証の入力バリデーション
2. エラーメッセージの多言語対応
3. ログイン試行回数の制限

要件：
- パスワードは8文字以上、大文字・小文字・数字を含む
- エラーメッセージは日本語と英語に対応
- 5回連続失敗でアカウントをロック
EOF

# ステップ 2: Claude Code にファイルを読ませる
claude "Please read /tmp/my-instructions.md and implement the feature described"
```

この方法は以下のメリットがあります：

- ターミナルやエディタの入力制限に影響されない
- ファイルとして保存されるため、再利用やバージョン管理が可能
- Claude Code がファイルを正確に読み取れる

## 5. Vim モード

Claude Code は Vim キーバインドのサブセットをサポートしています。Vim に慣れた開発者は、プロンプト入力をより効率的に操作できます。

### 有効化方法

```
/vim
```

または `/config` コマンドから設定画面を開いて有効化することもできます。一度有効にすると、次回以降のセッションでも Vim モードが維持されます。無効にしたい場合は再度 `/vim` を実行してください。

### サポートされているキーバインド

#### モード切替

| キー | 動作 |
|-----|------|
| `Esc` | NORMAL モードに切り替え |
| `i` | カーソル位置で INSERT モードに入る |
| `I` | 行頭で INSERT モードに入る |
| `a` | カーソルの右で INSERT モードに入る |
| `A` | 行末で INSERT モードに入る |
| `o` | 現在の行の下に新しい行を作成して INSERT モードに入る |
| `O` | 現在の行の上に新しい行を作成して INSERT モードに入る |

#### カーソル移動

| キー | 動作 |
|-----|------|
| `h` / `j` / `k` / `l` | 左 / 下 / 上 / 右 |
| `w` / `e` / `b` | 次の単語の先頭 / 単語の末尾 / 前の単語の先頭 |
| `0` / `$` | 行頭 / 行末 |
| `gg` / `G` | テキストの先頭 / 末尾 |
| `f` / `F` / `t` / `T` + 文字 | 文字の検索移動（`;` / `,` で次 / 前） |

#### 編集操作

| キー | 動作 |
|-----|------|
| `x` | カーソル位置の文字を削除 |
| `dw` / `de` / `db` | 単語を削除 |
| `dd` / `D` | 行を削除 / カーソルから行末まで削除 |
| `cw` / `ce` / `cb` / `cc` / `C` | 変更（削除して INSERT モード） |
| `.` | 直前の操作を繰り返す |

#### ヤンク / ペースト

| キー | 動作 |
|-----|------|
| `yy` / `Y` | 行をヤンク |
| `yw` / `ye` / `yb` | 単語をヤンク |
| `p` / `P` | カーソルの後ろ / 前にペースト |

#### テキストオブジェクト

| キー | 動作 |
|-----|------|
| `iw` / `aw` | 内側の単語 / 単語（空白含む） |
| `iW` / `aW` | 内側の WORD / WORD（空白含む） |
| `i"` / `a"` | ダブルクォート内 / ダブルクォート含む |
| `i'` / `a'` | シングルクォート内 / シングルクォート含む |
| `i(` / `a(` | 括弧内 / 括弧含む |
| `i[` / `a[` | ブラケット内 / ブラケット含む |
| `i{` / `a{` | 波括弧内 / 波括弧含む |

#### その他

| キー | 動作 |
|-----|------|
| `>>` / `<<` | インデント追加 / 削除 |
| `J` | 現在の行と次の行を結合 |

## ハンズオン演習

### 演習 1: 改行入力を設定する

**目的**: 自分のターミナルで最適な改行入力方法を確認し、複数行のプロンプトを送信する

**前提条件**: Claude Code がインストール済み、ターミナルが起動していること

**手順**:

1. Claude Code を起動する
   ```bash
   claude
   ```
2. まず `\` + Enter で改行を試す。以下のように入力する：
   ```
   以下の2つのファイルを作成してください\
   1. hello.py - Hello World を表示\
   2. goodbye.py - Goodbye を表示
   ```
3. 自分のターミナルが Shift+Enter に対応しているか確認する（iTerm2 / WezTerm / Ghostty / Kitty の場合は設定不要で試せる）
4. 対応していない場合は `/terminal-setup` を実行して自動設定する
5. Shift+Enter で改行が入ることを確認する

**期待される結果**: 複数行のプロンプトが正しく入力でき、1回の送信で Claude に伝わること

### 演習 2: デスクトップ通知を設定する

**目的**: Claude Code の処理完了通知が表示されるようにする

**前提条件**: macOS または Linux 環境

**手順**:

1. 自分のターミナルの通知対応状況を確認する
2. Kitty / Ghostty の場合：何もしなくてよい
3. iTerm2 の場合：
   - **Settings** → **Profiles** → **Terminal** → 通知センターアラートを有効化
   - アラートフィルタでエスケープシーケンスを有効化
4. macOS Terminal.app やその他の場合：Notification hooks を `.claude/settings.json` に追加する
5. Claude Code で長めの処理を実行し、通知が届くか確認する
   ```
   このプロジェクトの全ファイルを分析して、アーキテクチャの概要を教えてください
   ```

**期待される結果**: Claude Code が処理を完了すると、デスクトップ通知（またはカスタムサウンド）が発生すること

### 演習 3: Vim モードを試す

**目的**: Vim モードを有効化し、基本操作を体験する

**前提条件**: Claude Code が起動していること

**手順**:

1. `/vim` を実行して Vim モードを有効化する
2. 「Hello World のプログラムを作成して」と入力する（INSERT モード）
3. `Esc` を押して NORMAL モードに切り替える
4. `0` で行頭に移動し、`$` で行末に移動する
5. `dd` で行を削除する
6. `i` で INSERT モードに戻り、新しいプロンプトを入力する
7. `Esc` → `A` で行末に追記する
8. 入力が完了したら Enter で送信する

**期待される結果**: Vim キーバインドでプロンプト入力を操作でき、NORMAL / INSERT モードの切り替えが自在にできること

## よくある質問

**Q: Shift+Enter が動かないのですが、どうすれば良いですか？**
A: まず自分のターミナルがネイティブ対応しているか確認してください（iTerm2 / WezTerm / Ghostty / Kitty）。対応していない場合は Claude Code 内で `/terminal-setup` を実行します。VS Code、Alacritty、Zed、Warp は自動設定に対応しています。それ以外の場合は `\` + Enter を使ってください。

**Q: iTerm2 で通知が表示されません**
A: 3つのチェックポイントを確認してください。(1) iTerm2 の Settings → Profiles → Terminal で通知センターアラートが有効になっていること。(2) アラートフィルタでエスケープシーケンスによるアラートが有効になっていること。(3) macOS のシステム設定 → 通知で iTerm2 に通知権限が付与されていること。

**Q: Vim モードを無効にする方法は？**
A: `/vim` を再度実行するか、`/config` から設定を変更してください。

**Q: VS Code のターミナルで長い貼り付けが切り捨てられます**
A: VS Code のターミナルにはバッファ制限があります。長い指示はファイルに書き出して、Claude Code にそのファイルを読ませるファイルベースワークフローを使ってください。

**Q: `/config` で変更した設定はどこに保存されますか？**
A: `~/.claude/` ディレクトリ配下の設定ファイルに保存されます。CLI と VS Code 拡張機能で共有されます。

**Q: どのターミナルが Claude Code に最も適していますか？**
A: Shift+Enter と通知の両方がデフォルトで対応している Ghostty と Kitty が最も親和性が高いです。macOS で iTerm2 を使う場合は通知設定を行ってください。VS Code のターミナルは拡張機能と併用する場合に便利です。

## まとめ

この章で学んだ重要ポイント：

- テーマは `/config` コマンドで変更でき、ステータスラインもカスタマイズ可能
- 改行入力は `\` + Enter（全ターミナル共通）、Shift+Enter（iTerm2 / WezTerm / Ghostty / Kitty）、`/terminal-setup`（VS Code / Alacritty / Zed / Warp）の3つの方法がある
- Option+Enter を使うには、macOS Terminal.app は「Use Option as Meta Key」、iTerm2 と VS Code は「Esc+」の設定が必要
- Kitty と Ghostty はデスクトップ通知がデフォルトで対応、iTerm2 は設定が必要、その他は Notification hooks を使う
- 長い入力は直接貼り付けを避け、ファイルベースワークフローを使う
- `/vim` で Vim キーバインドを有効化でき、NORMAL / INSERT モードで操作できる

### ターミナルの選択ガイド

Claude Code を快適に使うためのターミナル選択の参考情報です：

| ターミナル | Shift+Enter | 通知 | 特徴 |
|-----------|------------|------|------|
| **iTerm2** | ネイティブ対応 | 設定必要 | macOS で最も人気。プロファイル管理が強力 |
| **WezTerm** | ネイティブ対応 | - | クロスプラットフォーム。Lua で設定可能 |
| **Ghostty** | ネイティブ対応 | デフォルト対応 | 高速レンダリング。通知もデフォルト対応 |
| **Kitty** | ネイティブ対応 | デフォルト対応 | GPU レンダリング。通知もデフォルト対応 |
| **VS Code** | `/terminal-setup` | - | IDE 統合。拡張機能との併用が便利 |
| **Alacritty** | `/terminal-setup` | - | 高速。ミニマルな設計 |
| **macOS Terminal** | `\`+Enter のみ | 非対応 | 追加インストール不要だが機能が限定的 |

通知とShift+Enter の両方がデフォルトで対応している **Ghostty** と **Kitty** が、Claude Code との親和性が最も高いターミナルです。
