## この章の学習目標

この章を終えると、以下のことができるようになります：

1. Web 版 Claude Code（claude.ai/code）にアクセスし、ブラウザからコーディングタスクを実行できる
2. `claude --remote` でターミナルから Web セッションを開始し、並列タスクを実行できる
3. `/teleport` コマンドで Web セッションをローカル環境に引き継げる
4. クラウド環境のツール構成とネットワーク設定を理解し、適切に活用できる

## 1. Web 版 Claude Code の概要

### Web 版とは

Web 版 Claude Code（[claude.ai/code](https://claude.ai/code)）は、ローカルに Claude Code をインストールすることなく、**ブラウザだけ**でコーディングタスクを実行できるサービスです。

タスクは **Anthropic が管理する仮想マシン**上で実行されるため、ローカル環境のセットアップは不要です。GitHub リポジトリのコードを分析・変更し、プルリクエストを作成するまでの一連の作業をブラウザ上で完結できます。

> **公式ドキュメントより**: 現在リサーチプレビュー段階です。

### 利用条件

| プラン | 利用可否 |
|-------|---------|
| **Pro** | 利用可能 |
| **Max** | 利用可能 |
| **Team** | 利用可能 |
| **Enterprise** | 利用可能（プレミアムシートまたは Chat + Claude Code シート） |

### Web 版が適しているシーン

| 用途 | 説明 |
|-----|------|
| **コード調査** | アーキテクチャや機能の実装方法の質問 |
| **バグ修正・ルーティンタスク** | 頻繁な操作不要の明確に定義されたタスク |
| **並列作業** | 複数のバグ修正を同時並行で実行 |
| **未チェックアウトのリポジトリ** | ローカルにクローンしていないコードでの作業 |
| **バックエンド変更** | テスト作成 → コード実装の TDD フロー |

## 2. はじめ方と動作の仕組み

### 初回セットアップ

1. [claude.ai/code](https://claude.ai/code) にアクセス
2. **GitHub アカウントを連携**
3. リポジトリに **Claude GitHub アプリ**をインストール
4. **デフォルト環境**を選択
5. コーディングタスクを送信
6. diff ビューで変更を確認し、反復してからプルリクエストを作成

### タスク実行の流れ

タスクを開始すると、以下のステップが自動で進行します：

```
1. リポジトリのクローン
   → Anthropic 管理の仮想マシンにリポジトリがクローンされる

2. 環境セットアップ
   → コードを含むセキュアなクラウド環境が準備される

3. ネットワーク設定
   → 設定に基づいてインターネットアクセスが構成される

4. タスク実行
   → コードの分析・変更・テスト実行・確認が行われる

5. 完了
   → ブランチにプッシュされ、PR 作成の準備が整う

6. 通知
   → 完了時に通知が届く
```

### diff ビューでの変更確認

Claude がファイルを変更すると、diff インジケーター（例: `+12 -1`）が表示されます。クリックすると diff ビューアが開きます：

- ファイルごとの変更をサイドバイサイドで確認
- 特定の変更にコメントして修正を依頼
- Claude との反復によって変更を磨く
- PR を作成する前に何度でも反復可能

## 3. ターミナルから Web へ（--remote フラグ）

ターミナルからコマンド一発で Web セッションを開始できます。

### 基本的な使い方

```bash
claude --remote "src/auth/login.ts の認証バグを修正して"
```

このコマンドは [claude.ai](https://claude.ai) に新しい Web セッションを作成します。タスクはクラウドで実行されるため、ローカルでは別の作業を続けられます。

### 複数タスクの並列実行

各 `--remote` コマンドは独立したセッションを作成するため、複数のタスクを並列で実行できます：

```bash
# 3つのタスクを並列で実行
claude --remote "auth.spec.ts のフレーキーなテストを修正して"
claude --remote "API ドキュメントを更新して"
claude --remote "ロガーを構造化出力に切り替えて"
```

`/tasks` コマンドでセッションの進捗を一覧確認できます。

### ローカルで計画、クラウドで実行

複雑なタスクでは、まずローカルで計画を立ててからクラウドで実行するパターンが効果的です：

```bash
# ステップ 1: ローカルでプランモードで計画を立てる
claude --permission-mode plan

# ステップ 2: 計画に満足したら、リモートセッションで実行
claude --remote "docs/migration-plan.md の移行計画を実行して"
```

## 4. Web からターミナルへ（/teleport）

Web セッションで作業した結果をローカル環境に引き継ぎたい場合、`/teleport` コマンドを使います。

### /teleport の使い方

Claude Code 内で実行：

```
/teleport
```

または略称 `/tp` で、Web セッションの一覧が表示されます。

コマンドラインから直接実行することもできます：

```bash
# Web セッション一覧を表示
claude --teleport

# セッション ID を直接指定
claude --teleport <session-id>
```

### /tasks からのテレポート

```
/tasks
```

でバックグラウンドセッション一覧を表示し、`t` キーを押して選択したセッションにテレポートできます。

### テレポートの前提条件

| 要件 | 詳細 |
|-----|------|
| **クリーンな Git 状態** | 未コミットの変更がないこと（ある場合はスタッシュを促される） |
| **正しいリポジトリ** | 同じリポジトリのチェックアウトから実行すること |
| **ブランチの存在** | Web セッションのブランチがリモートにプッシュされていること |
| **同じアカウント** | Web セッションと同じ claude.ai アカウントで認証済みであること |

## 5. モバイル対応

Claude iOS アプリおよび Android アプリから Web セッションを開始・監視できます。

### モバイルでできること

- 外出先からコーディングタスクを開始
- 進行中の作業をモニタリング
- ブラウザと同様に、セッション内で Claude に指示を送信
- 完了通知の受信

### アプリのダウンロード

Claude Code 内で `/mobile` コマンドを実行すると、ダウンロード用の QR コードが表示されます：

```
/mobile
```

## 6. クラウド環境とネットワークアクセス

### プリインストールされているツール

Web 版のクラウド環境には、主要な言語とツールがプリインストールされています：

| カテゴリ | 内容 |
|---------|------|
| **言語** | Python、Node.js、Ruby（3.1.6 / 3.2.6 / 3.3.6）、PHP 8.4.14、Java、Go、Rust、C++ |
| **パッケージマネージャ** | pip、poetry、npm、yarn、pnpm、bun、gem、bundler、Maven、Gradle、cargo |
| **データベース** | PostgreSQL 16、Redis 7.0 |

インストール済みのツールを確認するコマンド：

```bash
check-tools
```

### 依存関係の管理

カスタム環境イメージは現時点では非対応ですが、**SessionStart フック**を使ってセッション開始時にパッケージをインストールできます。

```json
// .claude/settings.json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "startup",
        "hooks": [
          {
            "type": "command",
            "command": "\"$CLAUDE_PROJECT_DIR\"/scripts/install_pkgs.sh"
          }
        ]
      }
    ]
  }
}
```

```bash
#!/bin/bash
# scripts/install_pkgs.sh

# リモート環境のみで実行（ローカルではスキップ）
if [ "$CLAUDE_CODE_REMOTE" != "true" ]; then
  exit 0
fi

npm install
pip install -r requirements.txt
exit 0
```

> **公式ドキュメントより**: `CLAUDE_CODE_REMOTE` 環境変数でリモート環境かどうかを判定できます。ローカルでは不要なインストールをスキップするのがベストプラクティスです。

### ネットワークアクセスの設定

| アクセスレベル | 説明 | デフォルト |
|-------------|------|-----------|
| **Full（制限なし）** | 全インターネットアクセス可能 | - |
| **Limited（制限あり）** | 許可ドメインのみ（npm、PyPI、GitHub 等） | デフォルト |
| **None（なし）** | インターネット接続なし | - |

デフォルトの「Limited」では、npm レジストリ、PyPI、GitHub など[許可ドメイン一覧](https://code.claude.com/docs/en/claude-code-on-the-web#default-allowed-domains)内のみアクセスできます。

### セキュリティ

- 各セッションは独立した Anthropic 管理の仮想マシンで実行
- Git 認証はスコープ限定のプロキシを通じて安全に処理
- 機密認証情報は Claude Code のサンドボックス内に存在しない

### 料金と制限

| 項目 | 詳細 |
|-----|------|
| **レート制限** | 通常の Claude Code 使用量と同じ |
| **並列実行** | 並列分のレート制限を消費 |
| **対応 VCS** | GitHub のみ（GitLab 等は非対応） |

### Web 版 vs ローカル CLI vs Remote Control の比較

| 観点 | Web 版 | ローカル CLI | Remote Control |
|-----|--------|------------|---------------|
| **セットアップ** | 不要 | CLI インストール必要 | CLI インストール必要 |
| **実行場所** | クラウド | ローカル | ローカル |
| **並列実行** | `--remote` で容易 | `--worktree` で可能 | 1セッション1接続 |
| **ローカルファイル** | アクセス不可 | 直接アクセス | 直接アクセス |
| **別デバイスから操作** | 常時可能 | 不可 | `/rc` で可能 |
| **オフライン動作** | 不可 | 可能 | ネット必要 |

## ハンズオン演習

### 演習 1: Web 版でコーディングタスクを実行する

**目的**: ブラウザから Claude Code を使い、コード変更と PR 作成の流れを体験する

**前提条件**: GitHub アカウント、Claude.ai の Pro / Max / Team / Enterprise プラン

**手順**:

1. [claude.ai/code](https://claude.ai/code) にアクセスする
2. GitHub アカウントを連携する（初回のみ）
3. 作業したいリポジトリを選択する
4. 「README.md にプロジェクトのセットアップ手順を追加して」と入力して送信する
5. タスクの進捗を確認する（ファイルの読み取り、変更、確認の各ステップ）
6. diff ビューで変更を確認する
7. 修正が必要ならコメントを追加して反復する
8. 満足したら PR を作成する

**期待される結果**: ブラウザだけで README.md の変更と PR 作成が完了すること

### 演習 2: --remote フラグで並列タスクを実行する

**目的**: ターミナルから複数の Web セッションを起動し、並列タスクを体験する

**前提条件**: Claude Code CLI がインストール済み、GitHub が連携済み

**手順**:

1. プロジェクトディレクトリに移動する
2. 以下の3つのコマンドを順に実行する：
   ```bash
   claude --remote "TODO コメントを整理して issue に変換して"
   claude --remote "未使用の import を削除して"
   claude --remote "テストカバレッジレポートを生成して"
   ```
3. `/tasks` コマンドで3つのセッションの進捗を確認する
4. 各セッションが独立して作業していることを確認する

**期待される結果**: 3つのタスクが並列で実行され、それぞれ独立した結果を返すこと

### 演習 3: /teleport で Web セッションをローカルに引き継ぐ

**目的**: Web セッションの結果をローカル環境にテレポートする

**前提条件**: 演習 1 または演習 2 で Web セッションが完了していること

**手順**:

1. ローカル環境で Git の作業ディレクトリがクリーン（未コミット変更なし）であることを確認する
2. Claude Code を起動する
   ```bash
   claude
   ```
3. `/teleport` を実行する
4. 表示される Web セッション一覧から引き継ぎたいセッションを選択する
5. ローカルのブランチが切り替わり、Web セッションの変更が反映されていることを確認する
6. ローカルで追加の変更やテストを実行する

**期待される結果**: Web セッションの変更がローカルブランチに反映され、そこから作業を継続できること

## よくある質問

**Q: Web 版とローカル CLI のどちらを使うべきですか？**
A: ローカル環境のセットアップなしで作業したい場合や、複数タスクを並列実行したい場合は Web 版が適しています。ローカルファイルへのアクセスや MCP サーバーが必要な場合は CLI を使ってください。

**Q: /teleport の前提条件「クリーンな Git 状態」とは？**
A: `git status` で未コミットの変更がない状態を指します。変更がある場合は、`git stash` でスタッシュするか、コミットしてからテレポートしてください。

**Q: GitLab や Bitbucket のリポジトリでは使えますか？**
A: 現時点では GitHub のみ対応です。GitLab 等の対応は今後予定されています。

**Q: クラウド環境にプリインストールされていないツールが必要な場合は？**
A: SessionStart フックを使って、セッション開始時に必要なツールをインストールできます。`CLAUDE_CODE_REMOTE` 環境変数で分岐させて、リモート環境でのみインストールを実行するスクリプトを `.claude/settings.json` に設定してください。

**Q: ネットワークアクセスを「Full」にしても安全ですか？**
A: 各セッションは独立した仮想マシンで実行され、機密認証情報はサンドボックス内に存在しません。ただし、外部サービスとの通信が必要な場合を除き、デフォルトの「Limited」を推奨します。

## まとめ

この章で学んだ重要ポイント：

- Web 版 Claude Code（claude.ai/code）はブラウザだけでコーディングタスクを実行できる
- `claude --remote "タスク"` でターミナルから Web セッションを開始し、並列実行が可能
- `/teleport`（または `/tp`）で Web セッションの結果をローカル環境に引き継げる
- クラウド環境には Python、Node.js、Go、Rust など主要言語がプリインストール済み
- SessionStart フックでカスタムの依存関係をインストールできる
- iOS / Android アプリからモバイルでの操作も可能
