## 1. Remote Control とは

### 概念

Remote Control は、**ローカルで実行中の Claude Code セッション**を、ブラウザやモバイルアプリから操作できる機能です。

例えば、デスクでタスクを開始してから外出し、電車の中でスマートフォンからそのまま作業を続ける、というシームレスなワークフローを実現します。

### なぜ Remote Control が必要か

従来、ローカルで実行中の開発タスクを別デバイスから続けるには、SSH 接続やリモートデスクトップなどの手段が必要でした。Remote Control はこれを Claude Code のレベルで解決し、Web ブラウザやモバイルアプリから直接セッションを操作できるようにします。

特に以下のような場面で威力を発揮します：

- 大規模なリファクタリングやテスト実行など、長時間かかるタスクの途中で離席する場合
- ローカルの MCP サーバーや特殊な開発環境に依存する作業を、別デバイスから指示したい場合
- デスクトップでの作業内容をそのままタブレットやスマートフォンに引き継ぎたい場合

### Web 版との決定的な違い

| 項目 | Remote Control | Web 版（claude.ai/code） |
|-----|---------------|------------------------|
| **実行場所** | ローカルマシン | Anthropic クラウド |
| **ローカルファイルアクセス** | あり（そのまま使える） | なし |
| **MCP サーバー** | ローカル設定を使用 | クラウド環境のみ |
| **ネット接続が切れた場合** | セッション中断（再接続可） | 継続実行 |
| **用途** | ローカル作業の遠隔操作 | ローカル環境不要の作業 |

**使い分けの指針**：

- **Remote Control**: 途中まで進めたローカル作業を別デバイスから続けたい場合。ローカルのファイルシステムや MCP サーバーがそのまま必要な場合。
- **Web 版**: ローカル設定なしで作業を始めたい場合。複数タスクを並列実行したい場合。

> **公式ドキュメントより**: Remote Control はローカルマシンで実行し続けます。Web 版（claude.ai/code）が Anthropic のクラウドインフラで実行されるのとは根本的に異なります。

## 2. 利用要件

Remote Control を使用する前に以下を確認してください：

| 要件 | 詳細 |
|-----|------|
| **サブスクリプション** | Max プラン（Pro プランのサポートは近日予定） |
| **認証方式** | API キーは非対応。`/login` で claude.ai 経由のサインインが必要 |
| **認証状態** | `claude` を起動して `/login` でサインイン済みであること |
| **ワークスペース信頼** | プロジェクトディレクトリで1回以上 `claude` を起動し、ワークスペース信頼ダイアログを承認済みであること |

> **公式ドキュメントより**: 現在、Remote Control は Max プランおよび Pro プランのリサーチプレビューとして提供されています。Team・Enterprise プランでは利用できません。

## 3. Remote Control セッションの開始

### 方法 1: 新規セッションとして開始

プロジェクトディレクトリに移動して実行します：

```bash
claude remote-control
```

コマンドを実行すると、ターミナルに以下が表示されます：

- **セッション URL**: 別デバイスからの接続に使用
- **QR コード**: スペースキーを押すと表示（スマートフォンからの接続に便利）
- **接続状態**: ツール活動のリアルタイム表示

#### オプション

| フラグ | 説明 |
|-------|------|
| `--verbose` | 接続とセッションの詳細ログを表示 |
| `--sandbox` | セッション中のファイルシステム・ネットワーク分離を有効化 |
| `--no-sandbox` | 分離を無効化（デフォルト） |

### 方法 2: 既存セッションを Remote Control に切り替える

既に Claude Code セッションを実行中の場合、そのセッションをそのまま Remote Control に切り替えられます：

```
/remote-control
```

略称：

```
/rc
```

現在の会話履歴を引き継いだまま、セッション URL と QR コードが表示されます。

> **公式ドキュメントより**: `/remote-control` を実行する前に `/rename` でセッション名を設定すると、デバイス間でセッションを見つけやすくなります。

## 4. 別デバイスからの接続

Remote Control セッションが開始されたら、以下のいずれかの方法で別デバイスから接続します。

### 方法 1: セッション URL をブラウザで開く

ターミナルに表示されている URL をコピーして、別デバイスのブラウザで開きます。[claude.ai/code](https://claude.ai/code) 上でセッションに直接アクセスできます。

### 方法 2: QR コードをスキャン

ターミナルで**スペースキー**を押すと QR コードが表示されます。スマートフォンのカメラでスキャンすると、Claude モバイルアプリが自動的に開いてセッションに接続します。

### 方法 3: セッション一覧から選択

[claude.ai/code](https://claude.ai/code) または Claude モバイルアプリを開き、セッション一覧から Remote Control セッションを探します。オンラインの Remote Control セッションには、**コンピューターアイコンに緑のステータスドット**が表示されます。

### 接続方法の比較

| 方法 | 最適なシーン | 手間 |
|-----|------------|------|
| **セッション URL** | PC 間の移動（URL をコピペできる環境） | 低 |
| **QR コード** | スマートフォンへの切り替え | 最低 |
| **セッション一覧** | 複数セッションがある場合の選択 | 中 |

スマートフォンへの切り替えでは QR コードが最も手軽です。PC 間では URL のコピー&ペーストが確実です。

## 5. 全セッションでの自動有効化

デフォルトでは、明示的に `claude remote-control` または `/remote-control` を実行した場合のみ Remote Control が有効になります。

すべてのセッションで自動的に有効にするには：

1. `/config` を起動する
2. 「**Enable Remote Control for all sessions**」を `true` に設定する

無効に戻す場合は `false` に設定します。

### 自動有効化の注意点

全セッションで Remote Control を有効にすると、常にリモート接続を待ち受ける状態になります。以下の点に注意してください：

- セキュリティ：不要な場合はオフにしておくことが推奨される
- リソース：ポーリングによる若干のネットワーク使用が発生する
- プライバシー：セッション一覧に全セッションが表示される

通常は必要な時だけ `/rc` で有効にする使い方が推奨されます。

## 6. セキュリティ

Remote Control の通信はセキュアに設計されています。

### 通信の仕組み

```
ローカルマシン                    Anthropic API                 別デバイス
(Claude Code)  ── HTTPS(出) ──>  (メッセージ  ──>  ブラウザ/
                                  ルーティング)    モバイルアプリ
```

### セキュリティの特徴

| 特徴 | 説明 |
|-----|------|
| **アウトバウンドのみ** | ローカルの Claude Code は HTTPS リクエストのみ発行。インバウンドポートは開かない |
| **ポーリング方式** | Remote Control 開始時に Anthropic API に登録し、定期的にポーリング |
| **TLS 暗号化** | 全トラフィックは TLS で暗号化された Anthropic API を経由 |
| **短命認証情報** | 複数の有効期限付き短命認証情報を使用。長期間の認証情報は保持しない |

## 7. ユースケース

### ユースケース 1: オフィスから在宅への切り替え

```
1. オフィスのターミナルで `claude remote-control` を実行
2. 帰宅後、自宅のブラウザやモバイルアプリからセッション URL にアクセス
3. 会話の続きから作業を再開
```

### ユースケース 2: ソファでのコードレビュー

```
1. デスクで `claude` セッション中に `/rc` を実行
2. タブレットのブラウザからセッションに接続
3. タブレットから変更の確認・フィードバックを送信
```

### ユースケース 3: 長時間タスクのモニタリング

```
1. 大規模リファクタリングのタスクを開始して `/rc` を実行
2. 外出先でモバイルアプリからセッションを確認
3. 必要に応じてモバイルから追加の指示を送信
4. 完了通知を受け取る
```

### ユースケース 4: ペアプログラミングの補助

```
1. 開発者 A がデスクで claude を起動して作業中
2. /rc を実行してセッション URL を開発者 B に共有
3. 開発者 B がブラウザからセッションに接続
4. 開発者 B がブラウザからレビューコメントや追加指示を送信
5. Claude がローカル環境で変更を実行
```

> **注意**: 同時に1つのリモート接続しかできないため、開発者 B だけがリモートから接続します。開発者 A はローカルのターミナルでセッションの状態を確認できます。

### Web 版・デスクトップ・Remote Control の選択フローチャート

どの機能を使うべきか迷ったときの判断基準：

```
ローカル環境が必要？
├── はい → Remote Control
│   └── 別デバイスから操作したい？
│       ├── はい → claude remote-control
│       └── いいえ → 通常の claude（CLI）
└── いいえ → Web 版 or デスクトップ
    └── 並列タスクを実行したい？
        ├── はい → Web 版（--remote）or デスクトップ（並列セッション）
        └── いいえ → Web 版 or デスクトップ（好みで選択）
```

## 8. 制限事項

| 制限 | 詳細 |
|-----|------|
| **同時接続数** | 1つの Claude Code セッションに1つのリモート接続のみ |
| **ターミナルの維持** | Remote Control はローカルプロセスとして実行。ターミナルを閉じるかプロセスを停止するとセッション終了 |
| **ネットワーク切断** | マシンが起動中でもネットワーク接続が約10分以上切断されるとタイムアウト。`claude remote-control` を再実行して新しいセッションを開始 |
| **プラン制限** | Max プランおよび Pro プランのみ。Team / Enterprise は現時点で非対応 |

### ターミナルを維持するための Tips

Remote Control はローカルプロセスとして動作するため、ターミナルが閉じるとセッションも終了します。長時間のセッションを維持するには：

- **tmux や screen を使う**: ターミナルマルチプレクサでセッションをデタッチ可能にする
- **macOS のスリープ設定**: システム設定 → ディスプレイ → スリープさせない を設定
- **caffeinate コマンド**: `caffeinate -i claude remote-control` でスリープを防止

```bash
# tmux で Remote Control を起動する例
tmux new-session -s claude-rc
claude remote-control
# Ctrl+B, D でデタッチ。後で tmux attach -t claude-rc で復帰
```

```bash
# caffeinate でスリープを防止
caffeinate -i claude remote-control
```

## ハンズオン演習

### 演習 1: Remote Control セッションを開始して別デバイスから接続する

**目的**: Remote Control の基本フローを体験する

**前提条件**: Max プランまたは Pro プラン、Claude Code CLI インストール済み、スマートフォンに Claude アプリインストール済み（またはブラウザが使える別デバイス）

**手順**:

1. プロジェクトディレクトリに移動する
2. `claude remote-control` を実行する
3. 表示されるセッション URL を確認する
4. スペースキーを押して QR コードを表示する
5. スマートフォンのカメラで QR コードをスキャンする（または別デバイスのブラウザでセッション URL を開く）
6. 別デバイスからプロンプトを送信する（例: 「このプロジェクトのファイル構成を教えて」）
7. ローカルのターミナルにツール活動がリアルタイムで表示されることを確認する

**期待される結果**: 別デバイスからの指示がローカルマシンで実行され、ファイルシステムにアクセスしたレスポンスが返ること

### 演習 2: 既存セッションを Remote Control に切り替える

**目的**: 進行中のセッションを別デバイスに引き継ぐフローを体験する

**前提条件**: Claude Code CLI インストール済み

**手順**:

1. ローカルで `claude` を起動する
2. いくつかの質問をして会話を進める（例: 「src/ ディレクトリの構成を教えて」）
3. `/rename my-session` でセッションに名前を付ける
4. `/rc` を実行して Remote Control に切り替える
5. 別デバイスの [claude.ai/code](https://claude.ai/code) でセッション一覧を確認する
6. 「my-session」という名前のセッション（緑のステータスドット付き）をクリックする
7. 会話履歴が引き継がれていることを確認する
8. 別デバイスから追加の指示を送信する

**期待される結果**: 会話履歴を含むセッションが別デバイスからそのまま操作でき、ローカルのファイルシステムにアクセスしたレスポンスが返ること

### 演習 3: Remote Control と Web 版の比較

**目的**: Remote Control と Web 版の違いを実際に体験して理解する

**前提条件**: 両機能が利用可能なプラン

**手順**:

1. Remote Control セッションで `ls -la` の結果を Claude に聞く（ローカルファイルが見えることを確認）
2. Web 版（claude.ai/code）でも同じリポジトリに対してセッションを開始する
3. 両方で「プロジェクトの構成を教えて」と質問する
4. 以下の違いを確認する：
   - Remote Control: ローカルファイルを直接読み取っている
   - Web 版: クラウド上のクローンを読み取っている
5. ローカルで新しいファイルを作成し、Remote Control では即座に見えるが、Web 版のセッションでは見えないことを確認する

**期待される結果**: Remote Control はローカルファイルシステムをリアルタイムに参照し、Web 版はクラウド上のスナップショットを参照していることが確認できること

## よくある質問

**Q: Remote Control と Web 版のどちらを使うべきですか？**
A: ローカルのファイルシステムや MCP サーバーにアクセスする必要がある場合は Remote Control。ローカル環境なしで作業したい場合やタスクを並列実行したい場合は Web 版を使ってください。

**Q: ターミナルを閉じてもセッションは維持されますか？**
A: いいえ。Remote Control はローカルプロセスとして実行されるため、ターミナルを閉じるとセッションは終了します。長時間の作業中にデバイスを移動する場合は、ターミナルを開いたまま（スリープさせずに）維持してください。

**Q: Team や Enterprise プランでは使えますか？**
A: 現時点では、Remote Control は Max プランおよび Pro プランのリサーチプレビューとして提供されています。Team・Enterprise プランでの対応は今後予定されています。

**Q: 接続が切断された場合はどうなりますか？**
A: 一時的な切断は自動的に再接続されます。ただし、ネットワーク接続が約10分以上切断されるとタイムアウトします。その場合は `claude remote-control` を再実行して新しいセッションを開始してください。

**Q: 複数のデバイスから同時に接続できますか？**
A: いいえ。1つの Claude Code セッションに対して1つのリモート接続のみです。別のデバイスから接続すると、先の接続は切断されます。

**Q: tmux や screen 内で Remote Control を使えますか？**
A: はい。`tmux new-session -s claude-rc` でセッションを作成し、その中で `claude remote-control` を実行してください。`Ctrl+B, D` でターミナルからデタッチしても Remote Control セッションは維持されます。

**Q: --sandbox オプションは何をしますか？**
A: `--sandbox` フラグを付けると、Remote Control セッション中のファイルシステムとネットワークの分離が有効になります。別デバイスからの操作でローカルファイルが意図せず変更されることを防ぎたい場合に使います。デフォルトは `--no-sandbox`（分離なし）です。

## 環境別アクセス方法の全体像

Week 3 で学んだ6つの環境と Remote Control の関係を整理します：

| 環境 | 実行場所 | リモートアクセス方法 |
|-----|---------|-------------------|
| ターミナル（CLI） | ローカル | Remote Control（`/rc`） |
| VS Code 拡張機能 | ローカル | Remote Control（`/rc`） |
| JetBrains プラグイン | ローカル | Remote Control（`/rc`） |
| デスクトップアプリ | ローカル | Remote Control（`/rc`） |
| Web 版 | クラウド | ブラウザ / モバイルアプリ（常時リモート） |
| Remote Control | ローカル（制御）+ リモート（操作） | セッション URL / QR / 一覧 |

Remote Control はローカルで実行中のすべての Claude Code セッション（CLI、VS Code、JetBrains、デスクトップアプリ内のターミナル）を対象にできます。

## まとめ

この章で学んだ重要ポイント：

- Remote Control はローカルで実行中のセッションをブラウザ / モバイルから操作する機能で、Web 版とは実行場所が根本的に異なる
- `claude remote-control` で新規セッション開始、既存セッションからは `/remote-control`（`/rc`）で切り替え
- セッション URL、QR コード、セッション一覧の3つの方法で別デバイスから接続できる
- 通信はアウトバウンド HTTPS + TLS + 短命認証情報でセキュアに設計されている
- ローカルのファイルシステムと MCP サーバーがそのまま使えるのが Web 版との最大の違い
