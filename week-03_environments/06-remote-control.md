# リモートコントロール

> 対応する公式ドキュメント: [Continue local sessions from any device with Remote Control](https://code.claude.com/docs/en/remote-control)

## 学習目標

- Remote Control の概念と Web 版との違いを理解する
- `claude remote-control` コマンドの使い方を習得する
- スマートフォン・タブレット・他のブラウザからの接続方法を理解する
- 外出先での継続作業のユースケースを把握する

## 概要

Remote Control は、ローカルで実行中の Claude Code セッションを、ブラウザやモバイルアプリから操作できる機能です。デスクでタスクを始めて、そのまま外出先でスマートフォンから作業を続ける、といったシームレスなワークフローを実現します。

**重要な違い**: Remote Control はローカルマシンで実行し続けます。Web 版（claude.ai/code）が Anthropic のクラウドインフラで実行されるのとは異なります。

> **注意**: Remote Control は現在、Max プランおよび Pro プランのリサーチプレビューとして提供されています。Team・Enterprise プランでは利用できません。

## Remote Control vs Web 版の違い

| 項目 | Remote Control | Web 版（claude.ai/code） |
|-----|---------------|------------------------|
| 実行場所 | ローカルマシン | Anthropicクラウド |
| ローカルファイルアクセス | あり | なし |
| MCP サーバー | ローカル設定を使用 | クラウド環境のみ |
| ネット接続が切れた場合 | セッション中断（再接続可） | 継続実行 |
| 用途 | ローカル作業の遠隔操作 | ローカル環境不要の作業 |

**使い分けの指針**:
- **Remote Control**: 途中まで進めたローカル作業を別デバイスから続けたい場合
- **Web 版**: ローカル設定なしで作業を始めたい、または複数タスクを並列実行したい場合

## 利用要件

Remote Control を使用する前に確認すること：

- **サブスクリプション**: Max プラン（Pro プランのサポートは近日予定）。API キーは非対応。
- **認証**: `claude` を起動して `/login` で claude.ai 経由でサインイン済みであること
- **ワークスペース信頼**: プロジェクトディレクトリで少なくとも1回 `claude` を起動して、ワークスペース信頼ダイアログを承認済みであること

## Remote Control セッションの開始

### 新規セッションとして開始

プロジェクトディレクトリに移動して実行：

```bash
claude remote-control
```

このコマンドはターミナルで実行し続け、リモート接続を待機します。表示される内容：

- セッション URL（別デバイスからの接続に使用）
- スペースキーを押すと QR コードが表示（スマートフォンからの接続に便利）
- 接続状態とツール活動のリアルタイム表示

オプション：

| フラグ | 説明 |
|-------|------|
| `--verbose` | 接続とセッションの詳細ログを表示 |
| `--sandbox` / `--no-sandbox` | セッション中のファイルシステム・ネットワーク分離を有効/無効化（デフォルト: 無効） |

### 既存セッションを Remote Control に切り替える

既に Claude Code セッションを実行中の場合、そのセッションを Remote Control に切り替えられます：

```
/remote-control
```

または略称：

```
/rc
```

現在の会話履歴を引き継いでセッション URL と QR コードが表示されます。

**ヒント**: `/remote-control` を実行する前に `/rename` でセッション名を設定すると、デバイス間でセッションを見つけやすくなります。

## 別デバイスからの接続

セッションが開始されると、以下の方法で接続できます：

### 方法1: セッション URL をブラウザで開く

ターミナルに表示されているURL をコピーして、別デバイスのブラウザで開きます。[claude.ai/code](https://claude.ai/code) 上でセッションに直接アクセスできます。

### 方法2: QR コードをスキャン

スペースキーを押して QR コードを表示し、スマートフォンのカメラでスキャンします。Claude モバイルアプリが自動的に開いてセッションに接続します。

### 方法3: セッション一覧から選択

[claude.ai/code](https://claude.ai/code) または Claude モバイルアプリを開き、セッション一覧から Remote Control セッションを探します。オンラインの Remote Control セッションには、コンピューターアイコンに緑のステータスドットが表示されます。

## 全セッションで Remote Control を有効化

デフォルトでは、明示的に `claude remote-control` または `/remote-control` を実行した場合のみ有効になります。全セッションで自動的に有効にするには：

1. `/config` を起動
2. **Enable Remote Control for all sessions** を `true` に設定

無効に戻す場合は `false` に設定します。

## セキュリティ

Remote Control の通信はセキュアに設計されています：

- ローカルの Claude Code セッションは**アウトバウンドの HTTPS リクエストのみ**を行う
- ローカルマシンでインバウンドポートは開かない
- Remote Control 開始時に Anthropic API に登録してポーリング
- 別デバイスから接続すると、サーバーがクライアントとローカルセッション間のメッセージをルーティング
- 全トラフィックは TLS で暗号化された Anthropic API を経由
- 複数の有効期限付き短命認証情報を使用

## ユースケース

### オフィスから在宅への切り替え

オフィスで作業中の Claude Code セッションをそのまま在宅に持ち帰る：

1. オフィスのターミナルで `claude remote-control` を実行
2. 帰宅後、自宅のブラウザやモバイルアプリからセッション URL にアクセス
3. 会話の継続から作業を再開

### ソファでのコードレビュー

デスクで Claude Code で実装して、リビングのタブレットでレビュー：

1. デスクで `claude` セッション中に `/remote-control` を実行
2. タブレットのブラウザからセッションに接続
3. タブレットから変更の確認・フィードバックを送信

### 長時間かかるタスクのモニタリング

ビルドや大規模なリファクタリングが完了するのを外出先からモニタリング：

1. タスクを開始して `/remote-control` を実行
2. 外出先でモバイルアプリからセッションを確認
3. 必要に応じてモバイルから追加の指示を送信

## 制限事項

| 制限 | 詳細 |
|-----|------|
| **同時接続数** | 1つの Claude Code セッションに1つのリモート接続のみ |
| **ターミナルを開いたまま** | Remote Control はローカルプロセスとして実行。ターミナルを閉じるかプロセスを停止するとセッション終了 |
| **長時間のネットワーク切断** | マシンが起動中でもネットワーク接続が約10分以上切断されるとタイムアウト。`claude remote-control` を再実行して新しいセッションを開始 |

## まとめ

- Remote Control はローカルで実行中のセッションをブラウザ/モバイルから操作する機能
- `claude remote-control` で新規 Remote Control セッションを開始
- 既存セッションからは `/remote-control`（または `/rc`）で切り替え
- セッション URL・QR コード・セッション一覧から別デバイスで接続できる
- Web 版と異なり、ローカルのファイルシステムや MCP サーバーがそのまま使える
