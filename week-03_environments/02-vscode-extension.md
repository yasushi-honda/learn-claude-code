# VS Code 拡張機能

> **対応公式ドキュメント**: https://code.claude.com/docs/en/vs-code
> **想定所要時間**: 約60分
> **難易度**: ★★☆☆☆
> **公式ドキュメント検証日**: 2026-03-26

## この章の学習目標

この章を終えると、以下のことができるようになります：

1. VS Code / Cursor に Claude Code 拡張機能をインストールし、パネルを開ける
2. @メンションでファイルやコード範囲をコンテキストとして共有できる
3. チェックポイント機能で変更の巻き戻しやフォークができる
4. 拡張機能の設定をカスタマイズし、CLI との違いを理解して使い分けられる

---

## 1. インストールと起動

### 前提条件

| 要件 | 詳細 |
|-----|------|
| **VS Code バージョン** | 1.98.0 以上 |
| **アカウント** | Anthropic アカウント（初回起動時にサインイン） |
| **Cursor** | Cursor エディタにも対応 |

> **公式ドキュメントより**: Amazon Bedrock や Google Vertex AI などのサードパーティプロバイダーを使用する場合は、別途認証設定が必要です。

### インストール手順

**方法 1: 拡張機能ビューから検索**

1. `Cmd+Shift+X`（Mac）または `Ctrl+Shift+X`（Windows/Linux）で拡張機能ビューを開く
2. 「**Claude Code**」を検索
3. **Install** をクリック

**方法 2: 直接リンクからインストール**

- VS Code: [vscode:extension/anthropic.claude-code](vscode:extension/anthropic.claude-code)
- Cursor: [cursor:extension/anthropic.claude-code](cursor:extension/anthropic.claude-code)

インストール後に拡張機能が表示されない場合は、VS Code を再起動するか、コマンドパレット（`Cmd+Shift+P`）から「**Developer: Reload Window**」を実行してください。

### Claude Code パネルを開く

Claude Code は「**Spark アイコン**」で識別されます。パネルを開く方法は3つあります：

| 方法 | 操作 | 備考 |
|-----|------|------|
| **エディタツールバー** | ファイル編集中にエディタ右上の Spark アイコンをクリック | 最も素早い方法 |
| **コマンドパレット** | `Cmd+Shift+P` → 「Claude Code」と入力 → 「Open in New Tab」等を選択 | 機能一覧が見える |
| **ステータスバー** | 画面右下の「Claude Code」をクリック | ファイルを開いていなくても使える |

## 2. プロンプトボックスと基本操作

### プロンプトの送信

プロンプトボックスに質問や指示を入力して **Enter** で送信します。Enter で送信してしまうのを避けたい場合は、設定の `claudeCode.useCtrlEnterToSend` を `true` にすると、`Ctrl+Enter`（Mac では `Cmd+Enter`）で送信するように変更できます。

### パーミッションモード

プロンプトボックスのモードインジケーターをクリックして、Claude の動作モードを切り替えられます：

| モード | 説明 |
|-------|------|
| **通常モード** | 各アクションの前に確認を求める |
| **プランモード** | 変更の計画を立てて承認後に実行 |
| **自動承認モード** | 編集を自動承認して確認なしで実行 |

### Extended thinking

拡張思考を有効にすると、Claude が複雑な問題に対してより深く推論します。プロンプトボックスの設定から切り替え可能です。

### コマンドメニュー

プロンプトボックスに `/` を入力すると、利用可能なコマンド（`/config`、`/vim`、`/plugins` など）の一覧が表示されます。CLI で使えるすべてのコマンドが利用可能なわけではないため、拡張機能で利用できるコマンドの一覧は `/` で確認してください。

### コンテキスト表示

Claude がどのファイルやコンテキストを参照しているかが、プロンプトボックスの上部に表示されます。

## 3. @メンションとファイル参照

### @メンションの使い方

`@` に続けてファイル名やフォルダ名を入力すると、特定のコンテキストを Claude に提供できます。**ファジーマッチ**に対応しているため、完全なファイル名を入力しなくても候補が表示されます。

```
> @auth の認証処理を説明して
  → auth.js、AuthService.ts、auth-middleware.ts 等にマッチ

> @src/components/ の構造を確認して
  → フォルダはスラッシュで終える

> @app.ts#5-10 のこの部分のバグを修正して
  → ファイル名#行番号-行番号 で特定の範囲を参照
```

### ショートカットで @メンション挿入

`Option+K`（Mac）/ `Alt+K`（Windows/Linux）を押すと、現在エディタで開いているファイルと選択範囲の @メンション参照（例: `@app.ts#5-10`）が自動的に挿入されます。

### ドラッグ&ドロップでファイル添付

**Shift キーを押しながらファイルをドラッグ&ドロップ**すると、ファイルを添付できます。画像ファイルも同様に添付可能です。

### @メンションの活用パターン

日常の開発でよく使う @メンション パターンをまとめます：

| パターン | 例 | 用途 |
|---------|-----|------|
| ファイル名のみ | `@auth.ts` | ファイル全体を参照 |
| 行範囲指定 | `@auth.ts#5-10` | 特定の行範囲を参照 |
| フォルダ指定 | `@src/components/` | フォルダ構造を参照 |
| ファジーマッチ | `@auth` | 部分一致するファイルを候補表示 |
| 複数参照 | `@auth.ts @user.ts` | 複数ファイルを同時に参照 |

## 4. 変更のレビューとチェックポイント

### インラインdiff

Claude がファイルを編集しようとすると、**サイドバイサイドの差分表示（diff）**が表示されます。変更内容を確認してから、以下の操作ができます：

- **Accept**: 変更を受け入れる
- **Reject**: 変更を拒否する
- テキストで別の指示を伝える

### チェックポイント（変更の巻き戻し）

会話の各ステップにチェックポイントが設定されます。メッセージにカーソルを**ホバー**すると巻き戻しボタンが表示されます：

| 操作 | 説明 |
|-----|------|
| **Fork**（会話をフォーク） | コード変更はそのままで、新しい会話分岐を開始 |
| **Rewind code**（コードを巻き戻し） | ファイル変更をこの時点に戻す（会話履歴は保持） |
| **Fork + Rewind**（フォーク+巻き戻し） | 新しい会話分岐を開始しつつ、ファイル変更も戻す |

> **公式ドキュメントより**: チェックポイントはファイルの状態のスナップショットです。Git のコミットとは異なり、Claude Code 内でのみ管理されます。

## 5. 会話管理とパネル配置

### 過去の会話の再開

パネル上部の**ドロップダウン**から過去の会話にアクセスできます：

- キーワード検索で絞り込み
- 期間フィルタ（今日、昨日、過去7日間など）
- **Remote タブ**でリモートセッションも表示可能

過去の会話をクリックすると、全メッセージ履歴付きで再開できます。

### 複数会話の並行実行

コマンドパレットから以下を実行して、複数の会話を並行して進められます：

- 「**Open in New Tab**」: 新しいエディタタブとして開く
- 「**Open in New Window**」: 新しいウィンドウとして開く

### パネルの配置

Claude Code パネルはさまざまな場所に配置できます：

| 配置場所 | 説明 |
|---------|------|
| **セカンダリサイドバー** | メインのサイドバーとは反対側に表示（推奨） |
| **プライマリサイドバー** | エクスプローラーと同じ側に表示 |
| **エディタ** | 通常のエディタタブとして表示 |

### ターミナルモード

VS Code 拡張機能の代わりに、ターミナルベースの CLI をそのまま VS Code 内で使いたい場合は、`claudeCode.useTerminal` 設定を `true` にします。この場合、グラフィカルパネルではなく、VS Code のターミナル内で CLI が起動します。CLI のすべてのコマンドとスキルが利用可能になる一方、ビジュアル diff やドラッグ&ドロップなどの拡張機能固有の機能は使えなくなります。

### どちらのモードを選ぶべきか

| 条件 | 推奨モード |
|-----|----------|
| ビジュアル diff やドラッグ&ドロップを使いたい | グラフィカルパネル（デフォルト） |
| CLI の全コマンドが必要（`!` ショートカット等） | ターミナルモード |
| 複数の拡張機能を並行して使いたい | グラフィカルパネル |
| Vim モードをプロンプト入力で使いたい | ターミナルモード |

## 6. 拡張機能の設定

### 主要な設定項目

`Cmd+,`（Mac）→「拡張機能」→「Claude Code」、または `settings.json` に直接記述します。

| 設定キー | デフォルト | 説明 |
|---------|-----------|------|
| `claudeCode.selectedModel` | `default` | 新しい会話で使用するモデル |
| `claudeCode.useTerminal` | `false` | グラフィカルパネルではなくターミナルモードで起動 |
| `claudeCode.initialPermissionMode` | `default` | 承認プロンプトの動作（`default` / `plan` / `acceptEdits` / `bypassPermissions`） |
| `claudeCode.preferredLocation` | `panel` | Claude を開く場所（`sidebar` / `panel`） |
| `claudeCode.autosave` | `true` | Claude が読み書きする前にファイルを自動保存 |
| `claudeCode.useCtrlEnterToSend` | `false` | Enter の代わりに Ctrl/Cmd+Enter でプロンプト送信 |

### Settings Schema サポート

`~/.claude/settings.json` にオートコンプリートと検証を有効にするには、`$schema` を追加します：

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json"
}
```

### VS Code コマンドとショートカット一覧

| コマンド | ショートカット（Mac） | ショートカット（Win/Linux） | 説明 |
|---------|---------------------|---------------------------|------|
| Focus Input | `Cmd+Esc` | `Ctrl+Esc` | エディタと Claude 間でフォーカスを切替 |
| Open in New Tab | `Cmd+Shift+Esc` | `Ctrl+Shift+Esc` | 新しい会話をエディタタブとして開く |
| New Conversation | `Cmd+N` | `Ctrl+N` | 新しい会話を開始（Claude にフォーカス時） |
| @-Mention | `Option+K` | `Alt+K` | 現在のファイルと選択範囲の参照を挿入 |

## 7. VS Code 拡張機能 vs CLI

| 機能 | CLI | VS Code 拡張機能 |
|-----|-----|-----------------|
| コマンドとスキル | 全て利用可能 | サブセットのみ（`/` で確認） |
| MCP サーバー設定 | CLI で設定 | CLI で設定、拡張機能で使用 |
| チェックポイント | 対応 | 対応 |
| `!` Bash ショートカット | 対応 | 非対応 |
| タブ補完 | 対応 | 非対応 |
| インラインdiff | ターミナル内 diff | IDE ネイティブ diff |
| ドラッグ&ドロップ | 非対応 | Shift+ドラッグで対応 |
| Chrome 連携 | 非対応 | `@browser go to localhost:3000` |

CLI のみの機能が必要な場合は、VS Code の統合ターミナルで `claude` を実行してください。

### Chrome 連携

VS Code 拡張機能では `@browser` を使って Chrome と連携できます：

```
> @browser go to localhost:3000
```

開発サーバーの画面を Claude に共有して、UI のフィードバックやデバッグの指示ができます。

### MCP 接続

MCP サーバーを HTTP トランスポートで追加する場合：

```bash
claude mcp add --transport http <server-name> <url>
```

設定は `~/.claude/settings.json` に保存され、拡張機能と CLI の両方で共有されます。

### Git Worktrees

VS Code 拡張機能でも Git worktree を使った並列作業が可能です：

```bash
claude --worktree
```

### プラグイン管理

プロンプトボックスに `/plugins` を入力すると、プラグイン管理インターフェースが開きます。

### アンインストール

拡張機能をアンインストールした後、関連データを完全に削除するには：

```bash
rm -rf ~/.vscode/globalStorage/anthropic.claude-code
```

### よく使う操作の早見表

| 操作 | ショートカット（Mac） | ショートカット（Win/Linux） |
|-----|---------------------|---------------------------|
| Claude にフォーカス | `Cmd+Esc` | `Ctrl+Esc` |
| 新しいタブで開く | `Cmd+Shift+Esc` | `Ctrl+Shift+Esc` |
| 新しい会話 | `Cmd+N` | `Ctrl+N` |
| @メンション挿入 | `Option+K` | `Alt+K` |
| コマンドパレット | `Cmd+Shift+P` | `Ctrl+Shift+P` |
| 拡張機能ビュー | `Cmd+Shift+X` | `Ctrl+Shift+X` |
| 設定画面 | `Cmd+,` | `Ctrl+,` |

## ハンズオン演習

### 演習 1: 拡張機能のインストールとパネル操作

**目的**: Claude Code 拡張機能をインストールし、3つの方法でパネルを開く

**前提条件**: VS Code 1.98.0 以上がインストール済み

**手順**:

1. `Cmd+Shift+X` で拡張機能ビューを開き、「Claude Code」を検索してインストール
2. 任意の `.ts` や `.py` ファイルを開き、エディタ右上の Spark アイコンをクリックしてパネルを開く
3. パネルを閉じてから、`Cmd+Shift+P` → 「Claude Code: Open in New Tab」で新しいタブとして開く
4. 画面右下のステータスバーから「Claude Code」をクリックして開く
5. Anthropic アカウントでサインインする

**期待される結果**: 3つの方法すべてで Claude Code パネルが開き、プロンプトが入力可能な状態になること

### 演習 2: @メンションとインラインdiff

**目的**: @メンションでコンテキストを共有し、Claude の変更を diff で確認する

**前提条件**: プロジェクトフォルダが開かれ、何らかのソースファイルが存在すること

**手順**:

1. エディタでソースファイルを開き、5-10行程度のコード範囲を選択する
2. `Option+K`（Mac）/ `Alt+K`（Win/Linux）を押して @メンション参照を挿入する
3. 「この関数にエラーハンドリングを追加してください」と入力して送信する
4. Claude が提案する変更を diff で確認する
5. 変更を Accept または Reject する
6. 別のファイル名を `@` で入力して、ファジーマッチの候補表示を確認する

**期待される結果**: @メンション参照が `@filename.ts#5-10` 形式で挿入され、Claude の変更が diff 表示されること

### 演習 3: チェックポイントの活用

**目的**: チェックポイント機能で変更を巻き戻す体験をする

**前提条件**: 演習 2 で Claude が何らかの変更を行っていること

**手順**:

1. Claude に追加の変更を依頼する（例: 「コメントを日本語で追加して」）
2. 変更が適用されたら、最初の変更のメッセージにカーソルをホバーする
3. 巻き戻しボタンが表示されることを確認する
4. 「**Rewind code**」をクリックして、コードを最初の変更前の状態に戻す
5. 会話履歴が保持されていることを確認する

**期待される結果**: ファイルの内容がチェックポイントの時点に戻り、会話履歴はそのまま残ること

## よくある質問

**Q: VS Code 拡張機能と CLI のどちらを使うべきですか？**
A: 普段 VS Code で開発しているなら拡張機能が便利です。IDE ネイティブの diff 表示、@メンション、ドラッグ&ドロップなどが使えます。ただし、CLI のみで利用可能なコマンド（`!` Bash ショートカット、タブ補完など）もあるため、必要に応じて VS Code の統合ターミナルで `claude` を起動してください。

**Q: Cursor エディタでも使えますか？**
A: はい、Cursor にも対応しています。インストール方法は VS Code と同じで、`cursor:extension/anthropic.claude-code` リンクからもインストールできます。

**Q: MCP サーバーの設定はどこで行いますか？**
A: MCP サーバーの設定は CLI で行い（`claude mcp add` コマンド）、設定は `~/.claude/settings.json` に保存されます。この設定は拡張機能と CLI の両方で共有されます。

**Q: Enter を押すと意図せず送信されてしまいます**
A: `claudeCode.useCtrlEnterToSend` を `true` に設定すると、Enter の代わりに `Ctrl+Enter`（Mac では `Cmd+Enter`）で送信するように変更できます。

**Q: サードパーティプロバイダー（Bedrock、Vertex AI）で使えますか？**
A: はい。拡張機能の設定画面またはコマンドラインで認証情報を設定してください。詳細は公式ドキュメントの「Third-party providers」セクションを参照してください。

**Q: 複数の会話を並行して進めるにはどうしますか？**
A: コマンドパレット（`Cmd+Shift+P`）から「Claude Code: Open in New Tab」を選択すると、新しいタブに独立した会話が開きます。「Open in New Window」で別ウィンドウとして開くこともできます。各会話は独立して動作します。

## まとめ

この章で学んだ重要ポイント：

- VS Code 1.98.0 以上が必要で、Cursor にも対応している
- パネルはエディタツールバー（Spark アイコン）、コマンドパレット、ステータスバーの3つの方法で開ける
- @メンションはファジーマッチ対応で、`@file.ts#5-10` 形式で行範囲も指定可能。`Option+K` / `Alt+K` で素早く挿入できる
- チェックポイントで Fork / Rewind code / Fork+Rewind の3種類の巻き戻しが可能
- 設定は `$schema` サポートで検証とオートコンプリートが使える
- CLI のみの機能が必要な場合は VS Code 統合ターミナルで `claude` を実行する
