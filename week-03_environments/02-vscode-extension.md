# VS Code 拡張機能

> 対応する公式ドキュメント: [Use Claude Code in VS Code](https://code.claude.com/docs/en/vs-code)

## 学習目標

- VS Code / Cursor に Claude Code 拡張機能をインストールする方法を覚える
- インラインdiff、@メンション、プランレビューなどの主要機能を理解する
- VS Code 固有のショートカットを習得する
- 設定のカスタマイズ方法を理解する
- CLI との違いを把握する

## 概要

VS Code 拡張機能は、Claude Code を VS Code に直接統合したグラフィカルインターフェースです。ターミナルを使わずに IDE 内で Claude とやり取りできます。インラインdiff での変更確認、@メンションによるファイル参照、プランのレビューと承認、会話履歴へのアクセスなど、多くの機能を備えています。

## 前提条件

- VS Code 1.98.0 以上
- Anthropic アカウント（初回起動時にサインイン）

> Amazon Bedrock や Google Vertex AI などのサードパーティプロバイダーを使用している場合は、別途設定が必要です。

## インストール

### VS Code へのインストール

1. `Cmd+Shift+X`（Mac）または `Ctrl+Shift+X`（Windows/Linux）で拡張機能ビューを開く
2. 「Claude Code」を検索
3. **インストール** をクリック

または、以下のリンクから直接インストールできます：
- [VS Code 向けインストール](vscode:extension/anthropic.claude-code)
- [Cursor 向けインストール](cursor:extension/anthropic.claude-code)

> インストール後に拡張機能が表示されない場合は、VS Code を再起動するか、コマンドパレットから「Developer: Reload Window」を実行してください。

## 基本的な使い方

### Claude Code パネルを開く

Claude Code は「Spark アイコン（✱）」で識別されます。パネルを開く方法は3つあります：

1. **エディタツールバー**: ファイルを開いているとき、エディタ右上に表示される Spark アイコンをクリック（最も素早い方法）
2. **コマンドパレット**: `Cmd+Shift+P` → 「Claude Code」と入力 → 「Open in New Tab」などを選択
3. **ステータスバー**: 画面右下の「✱ Claude Code」をクリック（ファイルを開いていなくても使用可能）

### プロンプトを送信する

プロンプトボックスに質問や指示を入力して Enter で送信します。

**ヒント**: テキストを選択すると Claude はその部分を自動的に認識します。`Option+K`（Mac）/ `Alt+K`（Windows/Linux）を押すと、選択部分の @メンション参照（例: `@file.ts#5-10`）が挿入されます。

### 変更をレビューする

Claude がファイルを編集しようとすると、変更前後の並列比較（diff）が表示され、承認を求められます。変更を受け入れる、拒否する、または別の指示を伝えることができます。

## 主要機能

### インラインdiff（変更のレビュー）

Claude が変更を加えるとき、サイドバイサイドの差分表示で変更内容を確認できます。各変更を個別に受け入れるか拒否することができます。

### @メンション（ファイル・フォルダの参照）

`@` に続けてファイル名やフォルダ名を入力することで、特定のコンテキストを Claude に提供できます。ファジーマッチングに対応しているため、部分的な名前でも検索できます：

```
> @auth の処理を説明して  （auth.js、AuthService.ts 等にマッチ）
> @src/components/ の構造を確認して  （フォルダはスラッシュを付ける）
```

### プランレビュー

複雑な変更を行う前に Claude が計画を立て、承認を求めます。計画を確認してから実行に移ることができます。

プロンプトボックスのモードインジケーターをクリックして以下のモードを切り替えられます：

| モード | 説明 |
|-------|------|
| **通常モード** | 各アクションの前に確認を求める |
| **プランモード** | Claude が変更の計画を立て、承認後に実行 |
| **自動承認モード** | 編集を自動承認して確認なしで実行 |

### 会話履歴

パネル上部のドロップダウンから過去の会話にアクセスできます。キーワード検索や期間（今日・昨日・過去7日間など）でフィルタリング可能です。過去の会話をクリックすると、全メッセージ履歴付きで再開できます。

### 複数会話の並行実行

コマンドパレットから「Open in New Tab」または「Open in New Window」を実行して、複数の会話を並行して進めることができます。

### チェックポイント（変更の巻き戻し）

メッセージにホバーすると巻き戻しボタンが表示されます：

- **そこから会話をフォーク**: コード変更はそのままで新しい会話分岐を開始
- **そこまでコードを巻き戻し**: ファイル変更をこの時点に戻す（会話履歴は保持）
- **会話フォークとコード巻き戻し**: 新しい会話分岐を開始しつつファイル変更を戻す

## VS Code コマンドとショートカット

コマンドパレット（`Cmd+Shift+P`）で「Claude Code」と入力すると、利用可能なコマンドが一覧表示されます。

| コマンド | ショートカット | 説明 |
|---------|-------------|------|
| フォーカス切替 | `Cmd+Esc` / `Ctrl+Esc` | エディタと Claude 間でフォーカスを切替 |
| 新しいタブで開く | `Cmd+Shift+Esc` / `Ctrl+Shift+Esc` | 新しい会話をエディタタブとして開く |
| 新しい会話 | `Cmd+N` / `Ctrl+N` | 新しい会話を開始（Claude にフォーカスが必要） |
| @メンション参照を挿入 | `Option+K` / `Alt+K` | 現在のファイルと選択範囲の参照を挿入 |

## 設定のカスタマイズ

### 拡張機能の設定

`Cmd+,`（Mac）または `Ctrl+,`（Windows/Linux）から「拡張機能 → Claude Code」で設定できます。

| 設定 | デフォルト | 説明 |
|-----|-----------|------|
| `selectedModel` | `default` | 新しい会話で使用するモデル |
| `useTerminal` | `false` | グラフィカルパネルではなくターミナルモードで起動 |
| `initialPermissionMode` | `default` | 承認プロンプトの動作（`default`、`plan`、`acceptEdits`、`bypassPermissions`） |
| `preferredLocation` | `panel` | Claude を開く場所（`sidebar` または `panel`） |
| `autosave` | `true` | Claude が読み書きする前にファイルを自動保存 |
| `useCtrlEnterToSend` | `false` | Enter の代わりに Ctrl/Cmd+Enter でプロンプト送信 |

### settings.json との連携

Claude Code の設定 `~/.claude/settings.json` は拡張機能と CLI で共有されます。許可コマンド、環境変数、フック、MCP サーバーの設定はここで行います。

オートコンプリートと検証を有効にするには、settings.json に以下を追加します：

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json"
}
```

## VS Code 拡張機能 vs CLI

| 機能 | CLI | VS Code 拡張機能 |
|-----|-----|-----------------|
| コマンドとスキル | 全て利用可能 | サブセットのみ（`/` で確認） |
| MCP サーバー設定 | CLI で設定 | CLI で設定、拡張機能で使用 |
| チェックポイント | 対応 | 対応 |
| `!` Bash ショートカット | 対応 | 非対応 |
| タブ補完 | 対応 | 非対応 |

CLI のみの機能が必要な場合は、VS Code の統合ターミナルで `claude` を実行します。

## プラグインの管理

プロンプトボックスで `/plugins` を入力すると、プラグイン管理インターフェースが開きます。インストール済みプラグインの有効/無効切替、マーケットプレイスからの新規インストールが可能です。

## まとめ

- VS Code 1.98.0 以上が必要
- Spark アイコン、コマンドパレット、ステータスバーからパネルを開ける
- @メンションでファイルやフォルダをコンテキストとして提供できる
- プランモードで変更前に計画をレビューできる
- 会話履歴から過去のセッションを再開できる
- チェックポイントで変更を巻き戻せる
- 設定は `settings.json` で拡張機能と CLI で共有
