# サブエージェントの活用

> 対応する公式ドキュメント: [Create custom subagents](https://code.claude.com/docs/en/sub-agents)

## 学習目標

- サブエージェントとは何か、どのような役割を持つかを理解する
- 組み込みサブエージェント（Explore、Plan、general-purpose）の特性を把握する
- カスタムサブエージェントを`.claude/agents/`ディレクトリに定義できる
- `--agents`フラグによる動的定義の方法を知る
- ツール制限・モデル設定・maxTurns・skills・mcpServersを適切に設定できる

## 概要

サブエージェントは、メインのClaude Codeセッションから生成される特化型のAIアシスタントです。各サブエージェントは独立したコンテキストウィンドウを持ち、カスタムシステムプロンプト、特定のツールアクセス権、独立したパーミッションで動作します。

タスクがサブエージェントの説明に合致すると、Claude Codeは自動的にそのサブエージェントへ委譲します。サブエージェントは独立して作業を行い、結果をメインセッションに返します。

**サブエージェントの主なメリット：**

- **コンテキスト保全**: 探索や実装作業をメイン会話から切り離してコンテキストを節約
- **制約の強制**: 特定のツールしか使えないよう制限することでセキュリティを強化
- **設定の再利用**: ユーザーレベルで定義してすべてのプロジェクトで共有
- **専門化**: ドメイン特化のシステムプロンプトで特定分野に集中
- **コスト制御**: 簡単なタスクはHaikuなど低コストモデルに委譲

> **注意**: サブエージェント内でさらにサブエージェントを生成することはできません（ネスト不可）。複数エージェントが互いに通信しながら協調作業する場合は [Agent Teams](02-agent-teams.md) を使用してください。

## 本文

### 組み込みサブエージェント

Claude Codeには以下の組み込みサブエージェントが含まれており、適切なタイミングで自動的に使用されます。

#### Explore

コードベースの検索・分析に特化した高速な読み取り専用エージェントです。

```
モデル: Haiku（高速・低レイテンシ）
ツール: 読み取り専用ツール（Write/Editは使用不可）
用途:  ファイル検索、コード検索、コードベース調査
```

Claude Codeがコードベースを理解する必要があるが変更は不要な場合に自動的に使用されます。調査結果をメインのコンテキストに持ち込まずに済むのが利点です。

#### Plan

プランモード中にコンテキストを収集するための調査エージェントです。

```
モデル: メイン会話から継承
ツール: 読み取り専用ツール
用途:  計画策定のためのコードベース調査
```

プランモードでコードベースを理解する必要がある場合に使用されます。サブエージェントはさらにサブエージェントを生成できないため（ネスト禁止）、この制約を安全に回避します。

#### general-purpose

探索と実装の両方が必要な複雑・多ステップタスク向けの汎用エージェントです。

```
モデル: メイン会話から継承
ツール: すべてのツール
用途:  複雑な調査、複数ステップの操作、コード変更
```

### カスタムサブエージェントの作成

#### `/agents`コマンドによる作成

`/agents`コマンドを実行すると対話型インターフェースが開きます：

```
/agents
```

- サブエージェントの一覧表示（組み込み・ユーザー・プロジェクト・プラグイン）
- 新規作成（ガイド付きセットアップ or Claude生成）
- 既存エージェントの編集
- 削除

#### ファイルによる手動定義

サブエージェントはYAMLフロントマター付きMarkdownファイルで定義します。

**ファイルの配置場所と優先度：**

| 場所 | スコープ | 優先度 |
|------|---------|--------|
| `--agents` CLIフラグ | 現在のセッションのみ | 1（最高） |
| `.claude/agents/` | 現在のプロジェクト | 2 |
| `~/.claude/agents/` | すべてのプロジェクト | 3 |
| プラグインの`agents/`ディレクトリ | プラグインが有効な場所 | 4（最低） |

**基本的なファイル形式：**

```markdown
---
name: code-reviewer
description: コードの品質とベストプラクティスをレビューする
tools: Read, Glob, Grep
model: sonnet
---

あなたはシニアコードレビュアーです。コードを分析し、品質・セキュリティ・ベストプラクティスについて
具体的でアクションに移せるフィードバックを提供してください。
```

#### フロントマターの設定フィールド

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `name` | 必須 | 小文字とハイフンのみの一意な識別子 |
| `description` | 必須 | Claudeがいつこのサブエージェントに委譲すべきか |
| `tools` | 任意 | 使用可能なツールの許可リスト（省略時は全ツール継承） |
| `disallowedTools` | 任意 | 拒否するツールのリスト |
| `model` | 任意 | `sonnet`、`opus`、`haiku`、`inherit`のいずれか（省略時は`inherit`） |
| `permissionMode` | 任意 | `default`、`acceptEdits`、`dontAsk`、`bypassPermissions`、`plan` |
| `maxTurns` | 任意 | 最大ターン数（超えると停止） |
| `skills` | 任意 | 起動時に注入するスキルのリスト |
| `mcpServers` | 任意 | このサブエージェントで利用可能なMCPサーバー |
| `hooks` | 任意 | このサブエージェント専用のライフサイクルフック |
| `memory` | 任意 | 永続メモリスコープ: `user`、`project`、`local` |
| `background` | 任意 | `true`にすると常にバックグラウンド実行 |
| `isolation` | 任意 | `worktree`にするとgit worktreeで分離実行 |

### モデルの選択

```yaml
model: sonnet   # Sonnet 4.6を使用
model: opus     # Opus 4.6を使用
model: haiku    # Haiku 4.5を使用
model: inherit  # メイン会話と同じモデル（デフォルト）
```

### ツール制限の設定

```yaml
---
name: safe-researcher
description: 読み取り専用の安全な調査エージェント
tools: Read, Grep, Glob, Bash
disallowedTools: Write, Edit
---
```

### `--agents`フラグによる動的定義

セッション開始時にCLIフラグでサブエージェントを定義することもできます：

```bash
claude --agents '{
  "code-reviewer": {
    "description": "コード変更後に積極的にレビューする専門家。",
    "prompt": "あなたはシニアコードレビュアーです。コード品質、セキュリティ、ベストプラクティスに集中してください。",
    "tools": ["Read", "Grep", "Glob", "Bash"],
    "model": "sonnet"
  }
}'
```

このフラグはセッション限定で、ファイルには保存されません。テストや自動化スクリプトに便利です。

### スキルのプリロード

サブエージェントにスキルをプリロードすることで、起動時からドメイン知識を持たせることができます：

```yaml
---
name: api-developer
description: チームの規約に従ってAPIエンドポイントを実装する
skills:
  - api-conventions
  - error-handling-patterns
---

APIエンドポイントを実装してください。プリロードされたスキルの規約とパターンに従ってください。
```

### 永続メモリの有効化

サブエージェントに会話をまたいで学習・記憶させることができます：

```yaml
---
name: code-reviewer
description: コードの品質とベストプラクティスをレビューする
memory: user
---

あなたはコードレビュアーです。コードをレビューしながら、発見したパターンや規約、
繰り返し発生する問題をエージェントメモリに記録してください。
```

| スコープ | 場所 | 用途 |
|---------|------|------|
| `user` | `~/.claude/agent-memory/<名前>/` | すべてのプロジェクト共通の学習 |
| `project` | `.claude/agent-memory/<名前>/` | プロジェクト固有・バージョン管理可能 |
| `local` | `.claude/agent-memory-local/<名前>/` | プロジェクト固有・ローカルのみ |

### フックによる条件制御

`PreToolUse`フックでツール使用前に検証を行えます。例えば読み取り専用DBクエリのみ許可するサブエージェント：

```yaml
---
name: db-reader
description: 読み取り専用のデータベースクエリを実行する
tools: Bash
hooks:
  PreToolUse:
    - matcher: "Bash"
      hooks:
        - type: command
          command: "./scripts/validate-readonly-query.sh"
---
```

検証スクリプト例（SQLの書き込み操作をブロック）：

```bash
#!/bin/bash
INPUT=$(cat)
COMMAND=$(echo "$INPUT" | jq -r '.tool_input.command // empty')

if echo "$COMMAND" | grep -iE '\b(INSERT|UPDATE|DELETE|DROP|CREATE|ALTER|TRUNCATE)\b' > /dev/null; then
  echo "ブロック: SELECT クエリのみ許可されています" >&2
  exit 2
fi

exit 0
```

### サブエージェントの実践例

#### コードレビュアー

```markdown
---
name: code-reviewer
description: コードの品質・セキュリティ・保守性を専門的にレビューする。コード変更後に積極的に使用する。
tools: Read, Grep, Glob, Bash
model: inherit
---

あなたはシニアコードレビュアーです。高い品質基準を維持します。

起動時の手順:
1. git diff で最近の変更を確認
2. 変更されたファイルに集中
3. すぐにレビューを開始

レビューチェックリスト:
- コードは明確で読みやすいか
- 関数名・変数名は適切か
- コードの重複はないか
- 適切なエラーハンドリングがあるか
- シークレットやAPIキーが露出していないか
- 入力バリデーションが実装されているか
- テストカバレッジは十分か
- パフォーマンス上の考慮がされているか

優先度別にフィードバックを提供する:
- 重大な問題（必ず修正）
- 警告（修正すべき）
- 提案（改善を検討）

問題の修正方法の具体例を示す。
```

#### デバッガー

```markdown
---
name: debugger
description: エラー・テスト失敗・予期しない動作のデバッグ専門家。問題に遭遇したら積極的に使用する。
tools: Read, Edit, Bash, Grep, Glob
---

あなたは根本原因分析の専門家です。

起動時の手順:
1. エラーメッセージとスタックトレースを取得
2. 再現手順を特定
3. 失敗箇所を特定
4. 最小限の修正を実装
5. 解決策を検証

デバッグプロセス:
- エラーメッセージとログを分析
- 最近のコード変更を確認
- 仮説を立てて検証
- 戦略的なデバッグログを追加
- 変数の状態を検査

各問題について提供する情報:
- 根本原因の説明
- 診断を裏付ける証拠
- 具体的なコード修正
- テストアプローチ
- 防止策の推奨
```

### フォアグラウンドとバックグラウンドの切り替え

```
# フォアグラウンドサブエージェント
# パーミッションプロンプトや質問がメインセッションに渡される

# バックグラウンドサブエージェント
# "バックグラウンドで実行して" と指示するか Ctrl+B を押す
```

バックグラウンドサブエージェントはメインセッションと並行して動作します。事前に必要なパーミッションの承認が求められ、承認後は自動的に実行されます。

### 特定のサブエージェントを無効化

`settings.json`で特定のサブエージェントの使用を禁止できます：

```json
{
  "permissions": {
    "deny": ["Agent(Explore)", "Agent(my-custom-agent)"]
  }
}
```

## まとめ

- サブエージェントは独立したコンテキストウィンドウを持つ特化型エージェントで、コンテキスト保全・ツール制限・専門化が主な利点
- 組み込みサブエージェント（Explore・Plan・general-purpose）は適切なタイミングで自動的に呼び出される
- `.claude/agents/`（プロジェクト）または`~/.claude/agents/`（ユーザー）に配置することでカスタムサブエージェントを定義できる
- `name`・`description`・`tools`・`model`・`maxTurns`・`skills`・`mcpServers`などのフロントマターフィールドで細かく設定できる
- サブエージェントはネストできない（サブエージェントからさらにサブエージェントを生成できない）
- フォアグラウンドとバックグラウンドの両モードで動作し、並列処理が可能
- サブエージェントに`memory`フィールドを設定すると会話をまたいで学習・記憶できる
