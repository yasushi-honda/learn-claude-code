# ADR-001: ヒーローバナーのアスペクト比ルール

## ステータス

採用 (2026-03-04)

## コンテキスト

各Week概要ページの上部にヒーローバナー画像（レッスンカード一覧のインフォグラフィック）を配置している。
画像はGemini 3.1 Flash Image APIで生成し、レッスン数に応じて1行または2行のカードレイアウトになる。

- Week 1-3: 5〜6レッスン → カード1行配置
- Week 4-8: 6〜7レッスン → カード2行配置

Gemini APIがサポートするアスペクト比は 1:1, 16:9, 9:16, 3:4, 4:3 のみ（21:9は非対応）。

## 決定

| パターン | 対象 | アスペクト比 | サイズ | 生成方法 |
|---------|------|------------|--------|---------|
| 1行レイアウト | Week 1-3 | 21:9 | 1376×589 | 16:9で生成 → PILで上部のみクロップ |
| 2行レイアウト | Week 4-8 | 16:9 | 1376×768 | 16:9で生成（そのまま使用） |

### クロップルール

```python
# 中央クロップ（アイコン上端とフッター両方を保持）
target_height = int(width * 9 / 21)
top = (height - target_height) // 2
cropped = img.crop((0, top, width, top + target_height))
```

### フロントエンド実装

```tsx
// site/src/app/week/[weekId]/page.tsx
<Image
  width={1376}
  height={week.number <= 3 ? 589 : 768}
/>
```

## 理由

- タイトルはテキストヘッダーで表示済みのため、画像にはレッスンカード+メタ情報のみを含める
- 1行レイアウトを16:9で表示すると上下の余白が大きくなり、バランスが悪い
- 21:9にすることでカード部分が画面幅いっぱいに広がり、視覚的に統一感が出る
- フッター情報（所要時間・レッスン数・難易度）とアイコン上端の両方を保持するため、クロップは中央基準

## 画像再生成時の注意

- Gemini APIのレート制限が厳しい（429エラー頻発）。生成間隔は60〜180秒を確保する
- プロンプトにタイトルを含めない（テキストヘッダーと重複するため）
- 生成後、Week 1-3はPILクロップが必要
