## 1. Claude Code in Slackとは

### 概要

Claude Code in Slackは、Slackワークスペースから`@Claude`メンションでコーディングタスクをWeb版Claude Codeに委譲できる機能です。チームの会話から直接、バグ修正・機能実装・コードレビューを開始できます。

```
┌─────────────────────────────────────────────────┐
│              Slack Channel/Thread               │
│                                                 │
│  User: @Claude バグを修正して                    │
│                ↓ コーディング意図を検出           │
│  Claude: Claude Codeセッションを作成中...         │
│           ↓                                     │
│  Claude: 作業中 (進捗更新)...                    │
│           ↓                                     │
│  Claude: 完了! [View Session] [Create PR]        │
└─────────────────────────────────────────────────┘
```

この連携は既存の「Claude for Slack」アプリをベースに構築されており、コーディング関連のリクエストをWeb上のClaude Codeにインテリジェントにルーティングする機能を追加しています。

### 主なユースケース

#### バグ調査と修正

バグがSlackチャンネルで報告されたら、すぐにClaudeに調査・修正を依頼できます。

```text
@Claude エラーログに「TypeError: Cannot read property 'id' of undefined」が出ています。
auth.tsのfindUserById関数を調査して修正してください
```

#### クイックコードレビューと修正

チームのフィードバックを元に、小さな機能追加やリファクタリングを依頼できます。

```text
@Claude チームのレビューで指摘された認証フォームの入力バリデーションを強化してください。
具体的には空白文字のみのメールアドレスをはじくようにしてほしいです
```

#### 協調デバッグ

スレッドの会話コンテキスト（エラー再現手順・ユーザーレポートなど）をClaudeが自動取得してデバッグに活用します。

```text
# スレッド内に既にエラー情報や再現手順がある場合
@Claude このスレッドの情報を使って原因を特定して修正してください
```

#### 並列タスク実行

Slackでタスクを起動して他の作業を続け、完了通知を受け取ります。

```text
@Claude 新しい通知システムのユニットテストを書いてください。
完了したら教えてください
```

## 2. 前提条件とセットアップ

### 前提条件

| 要件 | 詳細 |
|------|------|
| Claudeプラン | Pro・Max・Teams・Enterprise（Claude Codeアクセス付き） |
| Web版Claude Code | claude.ai/codeへのアクセスが必要 |
| GitHubアカウント | Web版Claude Codeに接続し、少なくとも1つのリポジトリを認証済み |
| Slack認証 | SlackアカウントをClaudeアカウントに連携済み |

> **重要**: Claude Code in SlackはGitHubのリポジトリのみ対応しています。GitLab等は現在未対応です。

### ステップ1: Slack App MarketplaceからClaude appをインストール

ワークスペースの管理者がSlack App MarketplaceからClaudeアプリをインストールします。

```
https://slack.com/marketplace/A08SF47R6P4
```

「Add to Slack」をクリックしてインストールを開始します。

> **注意**: Slackワークスペースへのアプリインストールには管理者権限が必要です。

### ステップ2: Claudeアカウントを接続する

各ユーザーが個人のClaudeアカウントを認証します。

1. Slackの「Apps」セクションで「Claude」をクリック
2. 「App Home」タブに移動
3. 「Connect」をクリックしてSlackアカウントとClaudeアカウントを連携
4. ブラウザで認証フローを完了

### ステップ3: Web版Claude Codeを設定する

1. [claude.ai/code](https://claude.ai/code)にSlackと同じアカウントでサインイン
2. GitHubアカウントを接続（未接続の場合）
3. 作業対象のリポジトリを少なくとも1つ認証

### ステップ4: ルーティングモードを選択する

Claude App HomeでRouting Modeを設定します。

| モード | 動作 | 推奨シーン |
|--------|------|----------|
| **Code only** | すべての@メンションをClaude Codeセッションにルーティング | Slackを開発タスク専用に使うチーム |
| **Code + Chat** | メッセージを分析してCode（コーディング）またはChat（質問・分析・執筆）に振り分け | Slackを様々な用途に使うチーム |

> **ヒント**: Code + Chatモードで誤ったルーティングが行われた場合:
> - ChatにルーティングされたがCodeが必要 → 「Retry as Code」をクリック
> - CodeにルーティングされたがChatが必要 → スレッドでChat選択

### ステップ5: チャンネルにClaudeを追加する

Claudeは自動的にどのチャンネルにも追加されません。使用したいチャンネルで招待します。

```
/invite @Claude
```

> **重要**: Claudeは招待されたチャンネルの`@Claude`メンションにのみ応答します。ダイレクトメッセージ（DM）では動作しません。

## 3. 動作の仕組み

### 自動検出とルーティング

`@Claude`をメンションすると、Claudeがメッセージを分析してコーディングタスクかどうかを判定します（Code + Chatモードの場合）。

```
1. @Claude でコーディングリクエストをメンション
2. Claudeがコーディング意図を検出
3. claude.ai/code上に新しいClaude Codeセッションを作成
4. 作業進捗がSlackスレッドに投稿
5. 完了時: @メンションで要約とアクションボタンを通知
6. [View Session]または[Create PR]でアクションを実行
```

### コンテキスト収集

**スレッド内でメンション**: スレッド内のすべてのメッセージからコンテキストを収集します。バグ報告スレッドに再現手順や詳細情報があれば、Claudeが自動的に参照します。

**チャンネルで直接メンション**: チャンネルの最近のメッセージから関連コンテキストを取得します。

> **セキュリティ注意**: `@Claude`を呼び出すと、Claudeは会話コンテキストにアクセスします。コンテキスト内の他のメッセージの指示に従う場合があるため、信頼できるSlack会話でのみ使用してください。

## 4. UIとアクションボタン

### セッション完了後のアクション

| ボタン | 機能 |
|--------|------|
| **View Session** | ブラウザでClaude Codeセッションの全履歴を表示。続きの指示も可能 |
| **Create PR** | セッションの変更からプルリクエストを直接作成 |
| **Retry as Code** | Chatとして応答された場合にClaude Codeセッションとして再試行 |
| **Change Repo** | Claudeが誤ったリポジトリを選択した場合に変更 |

### リポジトリ選択

Claudeは会話コンテキストから自動的にリポジトリを選択します。複数のリポジトリが当てはまる場合はドロップダウンで選択できます。リポジトリ名をメッセージに明示すると精度が向上します。

```text
@Claude myorg/api-serverリポジトリのログイン関数のバグを修正してください
```

## 5. ワークフロー例

### バグ報告 → PR作成フロー

```
# ステップ1: バグがチャンネルで報告される
ユーザーA: 本番環境でログイン時に500エラーが出ています
           ログ: "UnhandledPromiseRejection: db timeout"

# ステップ2: Claudeに委譲
ユーザーB: @Claude このスレッドを見て、DBタイムアウトの問題を調査して修正してください

# ステップ3: Claudeが作業
Claude: リポジトリを分析しています...
Claude: database.tsのconnectionPool設定を確認中...
Claude: 修正完了! タイムアウト設定を30秒に延長しました

# ステップ4: PRを作成
Claude: 作業が完了しました。[View Session] [Create PR]
→ [Create PR]クリックでPRが自動作成
```

### 機能実装 → テスト作成

```text
@Claude payments/stripe.tsに支払い失敗時のリトライロジックを追加してください。
最大3回リトライし、指数バックオフを使用してください。
完了したらテストも書いてください
```

### SlackとWeb版の使い分け

| Slackを使う場面 | Web版を直接使う場面 |
|----------------|-------------------|
| バグ報告やSlackの議論コンテキストが既にある | ファイルをアップロードしたい |
| 非同期でタスクを委譲したい | リアルタイムでやりとりしたい |
| チームに作業の可視性が必要 | 長期・複雑なタスクに取り組む |

## 6. アクセスとパーミッション管理

### ユーザーレベルのアクセス

| アクセス種別 | 要件 |
|------------|------|
| Claude Codeセッション | 各ユーザーが自分のClaudeアカウントでセッションを実行 |
| 使用量・レート制限 | 個人プランの制限に対してカウント |
| リポジトリアクセス | 個人がWeb版Claude Codeに接続したリポジトリのみ |
| セッション履歴 | claude.ai/codeのClaude Code履歴に表示 |

### ワークスペースレベルの管理

| 制御 | 説明 |
|------|------|
| アプリインストール | ワークスペース管理者がClaude appの導入を決定 |
| Enterprise Grid | 組織管理者がどのワークスペースがアクセスできるか制御 |
| アプリ削除 | ワークスペースからアプリを削除すると全ユーザーのアクセスが即座に無効化 |

### チャンネルベースのアクセス制御

チャンネルへの招待によってアクセスを細かく制御できます。

```
# Claudeを特定チャンネルに追加
/invite @Claude

# 開発チームのみのプライベートチャンネルを活用
# → #dev-claudeなどの専用チャンネルでアクセスを制限
```

- **招待が必要**: Claudeが追加されたチャンネルでのみ動作
- **プライベートチャンネル対応**: パブリック・プライベート両方で利用可能
- **管理者によるゲーティング**: Claudeを招待するチャンネルを管理することで使用範囲を制限

## 7. 効果的な使い方のベストプラクティス

### 具体的なリクエストを書く

```text
# 良い例: 具体的なファイルと関数を指定
@Claude src/auth/login.ts のhandleLoginError関数に、
ネットワークエラーと認証エラーを区別するロジックを追加してください

# 悪い例: 曖昧で広範
@Claude ログインを修正してください
```

### スレッドのコンテキストを活用する

```text
# スレッド内のエラーログ・再現手順・チームの議論 → すべてClaudeが参照
@Claude このスレッドの情報を使って原因を特定して修正してください
```

### 完了の定義を明確にする

```text
@Claude ユーザー検索APIのパフォーマンスを改善してください。
テストを書いて、ドキュメントを更新して、PRを作成するまでが完了です
```

### リポジトリ名を明示する

複数のリポジトリがある場合は明示的に指定すると精度が向上します。

```text
@Claude myorg/api-server リポジトリのログイン関数のバグを修正してください
```

## 8. セキュリティの考慮事項

### コンテキストのセキュリティ

`@Claude`を呼び出すと、チャンネルまたはスレッドの会話コンテキストをClaudeが読み取ります。

1. **信頼できるチャンネルでのみ使用**: 外部ユーザーが参加しているチャンネルでは、プロンプトインジェクション攻撃のリスクがある
2. **機密情報に注意**: パスワードやAPIキーが含まれるスレッドでは使用を避ける
3. **プライベートチャンネルを活用**: 機密性の高いコードベースには専用のプライベートチャンネルを作成する

### Teams/Enterpriseのセッション共有

TeamsまたはEnterpriseアカウントでは、Slackから作成されたセッションが組織内で自動的に可視化されます。

### 現時点の制限事項

| 制限 | 内容 |
|------|------|
| GitHub限定 | GitLabなど他のGitプラットフォームは未対応 |
| PR作成は1件 | 各セッションで作成できるPRは1つ |
| レート制限 | 個人のClaudeプランのレート制限が適用 |
| Webアクセスが必要 | Web版Claude Codeのアクセスがないユーザーは通常のClaude Chatのみ |
| DMは非対応 | チャンネルでのみ動作 |

## よくある質問

**Q: Claude Code in Slackは無料プランで使えますか?**
A: いいえ。Pro・Max・Teams・Enterpriseのいずれかのプランが必要で、かつWeb版Claude Codeへのアクセスが必要です。

**Q: DMでClaudeにコーディングタスクを送れますか?**
A: いいえ。Claude Code in Slackはチャンネル内の`@Claude`メンションにのみ対応しています。DMでは通常のClaude Chat応答のみです。

**Q: GitLabのリポジトリは使えますか?**
A: 現時点ではGitHubリポジトリのみ対応しています。GitLab対応は将来的に追加される可能性があります。

**Q: セッションのレート制限はどのように適用されますか?**
A: 各ユーザーの個人Claudeプランのレート制限が適用されます。チーム全体で共有される制限はありません。

**Q: Claudeが間違ったリポジトリを選択した場合はどうすればいいですか?**
A: 完了通知に表示される「Change Repo」ボタンをクリックして変更できます。また、リクエストにリポジトリ名を明示（例: `@Claude myorg/api-server の...`）すると精度が向上します。

## まとめ

この章で学んだ重要ポイント：

- Claude Code in Slackは`@Claude`メンションでコーディングタスクをWeb版Claude Codeセッションに委譲する機能
- セットアップにはClaude appのインストール、アカウント連携、GitHub接続、チャンネル招待が必要
- ルーティングモードは「Code only」（コーディング専用）と「Code + Chat」（インテリジェント振り分け）の2種類
- スレッドの会話コンテキストを自動収集して、バグ報告からPR作成まで一連のフローを自動化できる
- チャンネルへの招待でアクセスを制御し、信頼できる環境でのみ使用する
- 現在はGitHubリポジトリのみ対応、DMは非対応
